# 03. 文献综述与文献定位

文献综述不只是"列文献"。它的核心功能是：在已有研究的版图上为本研究找到位置，并论证本研究的必要性。

---

## 一、文献综述的三重功能

1. **展示版图**：让读者看到本研究所属领域的现状
2. **找到缺口**：识别本研究填补的空白
3. **建立对话**：把本研究嵌入正在进行的学术讨论

只完成第一项的文献综述是"文献堆砌"，C 刊不接受。

---

## 二、文献搜集的三个层次

### 2.1 核心文献（5-10 篇）

与本研究问题最直接相关、必须吃透的文献。这些文献在论文中会被多次引用，并构成论证的关键支撑。

### 2.2 重要文献（15-25 篇）

与本研究相关、需要熟悉但不必逐字研读的文献。在文献综述节会被引用，但正文中可能只引一次。

### 2.3 背景文献（25-50 篇）

提供研究背景的文献，包括理论经典、政策文件、行业报告等。在背景介绍、理论框架、方法说明等部分被引用。

中文 C 刊投稿，参考文献总数通常 30-50 条之间。少于 25 条会被认为对文献吃得不深；多于 60 条会被认为堆砌。

---

## 三、中英文献的比例

C 刊投稿不要做成"全是英文文献"或"全是中文文献"：

| 文献类型 | 推荐比例（参考） |
|---|---|
| 中文学术期刊论文 | 30-50% |
| 英文学术期刊论文 | 20-35% |
| 中英学术专著 | 10-20% |
| 政策文件与一手资料 | 10-25% |
| 行业报告与新闻 | ≤ 10% |

### 3.1 中文期刊文献尤其重要

中文 C 刊审稿人最敏感的是"对本国学术社群的对话"。如果一篇论文几乎全引英文文献和西方理论，会被判定为"脱离本土学术语境"。

解决方法：
- 在 CNKI 搜索本研究主题，找出近 5 年发表在目标期刊及同档期刊的论文
- 至少引用 5-8 篇相关的中文期刊论文
- 在文献综述部分明确与中文学者的对话

### 3.2 目标期刊自身的论文

强烈建议引用 2-3 篇目标期刊近 3 年的相关论文。这表明作者对该刊学术取向有了解，也提升录用概率。

但**不要硬塞**——如果目标期刊近期没有相关主题论文，宁可不引，也不要找牵强的连接。

---

## 四、文献综述的写法

### 4.1 三种常见结构

#### 结构 A：按主题分块

```
## X. 文献综述

### X.1 主题 A 的研究进展
### X.2 主题 B 的研究进展
### X.3 主题 C 的研究进展
### X.4 已有研究的不足与本研究的位置
```

适合：研究问题涉及多个学术议题的整合。

#### 结构 B：按理论传统分块

```
## X. 文献综述

### X.1 理论传统 A 的视角及研究
### X.2 理论传统 B 的视角及研究
### X.3 两种视角的对话与本研究的整合
```

适合：研究涉及理论对话或框架整合。

#### 结构 C：按时间或阶段分块

```
## X. 文献综述

### X.1 早期研究（描述阶段）
### X.2 中期研究（解释阶段）
### X.3 近期研究（机制阶段）
### X.4 研究缺口
```

适合：研究领域有清晰的演进脉络。

### 4.2 不应使用的写法

- **罗列式**："X 学者认为……Y 学者指出……Z 学者主张……"——这是 综述报告的写法，不是学术论文的文献综述
- **过度展开式**：每篇文献都展开 200-300 字——文献综述节会过长
- **无判断式**：只描述别人写了什么，不给评价——读者看不出本研究的位置

### 4.3 应使用的写法

- **概念地图式**：用 1-2 段把领域内的核心概念和争论说清楚
- **演进脉络式**：用 1-2 段说清楚领域内的发展阶段或范式转换
- **学派对话式**：用 1-2 段说清楚不同学派的争论焦点
- **缺口聚焦式**：用 1 段明确指出哪些问题尚未充分研究

---

## 五、文献定位声明

文献综述节末尾必须有一段"文献定位声明"，明确说明本研究在已有研究中的位置。模板：

```
已有研究在以下方面取得了重要进展：第一，……；第二，……；第三，……。
但仍有以下问题尚未充分讨论：第一，……；第二，……。
本研究在以下方面对已有研究有所推进：第一，……；第二，……。
```

注意：
- 不要说"国内研究不足，国外研究丰富"——这是常见的偷懒说法
- 不要说"已有研究多以 X 为对象，本研究以 Y 为对象"——研究对象的差异不构成贡献
- 应该说在某个具体的概念、机制、解释、方法上推进了什么

## 六、文献核实

### 6.1 中文文献核实

通过 CNKI、Wanfang 检索条目核实：
- 卷期页码（最易出错）
- 作者列表（多作者文献容易漏列）
- 题目（容易抄错）

不确定时标注"可能不准确，建议在 CNKI 核实"，不要编。

### 6.2 英文文献核实

通过 Google Scholar、Web of Science、出版社官网检索：
- DOI（优先标注）
- 期刊全称（不缩写）
- 卷期号 + 页码

### 6.3 政策文件核实

- 政府门户网站为权威源
- 链接易失效，引用时注明"如链接失效，可在 X 检索关键词 Y"
- 标注访问日期

### 6.4 BAI（韩国审计院）等特殊源

- BAI 禁止自动抓取，需手动验证 srno
- 链接 URL pattern 为 `?srno=XXXX`

---

## 七、文献的二次引用

避免二次引用（即"X 引自 Y"）。如果某个观点对论证重要，应找到原始文献阅读。

例外：经典文献且无法获取原文时，可标注"转引自 X"，但应控制在 2-3 处以内。

---

## 八、文献综述的字数控制

文献综述节通常 1500-3000 字。具体取决于：

| 论文总字数 | 文献综述节字数 |
|---|---|
| 8000-12000 字（短论文） | 1000-1500 字 |
| 12000-15000 字（标准论文） | 1500-2500 字 |
| 15000-20000 字（长论文） | 2000-3500 字 |

超过总字数 20% 的文献综述节是过度。

---

## 九、文献综述与理论框架的关系

两者经常被混淆。区别：

| 维度 | 文献综述 | 理论框架 |
|---|---|---|
| 功能 | 展示已有研究、找到缺口 | 提供分析工具 |
| 内容 | 同主题的实证研究 | 理论概念与逻辑 |
| 位置 | 通常在引言后 | 通常在文献综述后 |
| 关键问题 | 已有研究做了什么、没做什么 | 用什么工具分析 |

如果论文短，可以合并为"文献综述与理论框架"一节；论文长则分开。

## 十、文献综述节的常见问题与诊断

### 10.1 文献综述节读起来像综述报告

**症状**：每段都是"X 学者……Y 学者……"
**诊断**：缺少综合性的概念地图或脉络
**修复**：把段落主语从"X 学者"换成"研究焦点""争论核心"或具体概念

### 10.2 文献综述节缺少对话

**症状**：列了大量文献，但看不出本研究和它们的关系
**诊断**：缺少文献定位声明
**修复**：在节末加一段定位声明

### 10.3 文献综述节有"硬伤"

**症状**：某重要相关文献未被引用
**诊断**：文献搜集不全
**修复**：用 CNKI、Google Scholar 重新搜索本主题的代表性文献

### 10.4 文献综述节用了无关文献

**症状**：列了一些和本研究关系不大的文献
**诊断**：为了"显得文献丰富"而硬塞
**修复**：删除。文献综述的质量在精不在多

---

## 十一、修订环节的文献检查

每轮修订都检查：

- [ ] 文献综述节是否还和当前研究问题对齐？
- [ ] 是否有新发表的重要文献需要补充？
- [ ] 文献定位声明是否还准确？
- [ ] 中文 C 刊文献占比是否合适？
- [ ] 引用了的文献是否都在文末参考文献中？（避免引而未列）
- [ ] 参考文献中是否有未在正文引用的？（避免列而未引）

引用与文献列表的对齐检查见 `B-references-gbt7714.md`。
