# J — 投稿前合规自检清单

本手册基于真实投稿项目的返工经验整理。核心教训：**格式合规问题在投稿后发现等于重做一遍**，在阶段 7 之前做完以下检查，成本为零。

---

## J.1 题目

| 检查项 | 操作 |
|---|---|
| 字数是否在目标期刊限制内 | 查期刊官网"投稿须知"，不同刊限制差异大（20字/25字/不限） |
| 副标题是否合规 | 部分刊禁止副标题；使用破折号连接副标题须符合该刊惯例 |
| 是否含有期刊禁止使用的词 | 少数刊禁止"研究""探讨"等词出现在题目中，需逐刊核查 |

---

## J.2 摘要

**高频返工原因：未按期刊要求的人称、禁用词和结构规范写摘要。**

### J.2.1 人称与措辞

以下词汇在多数 C 刊摘要中被明确禁用，**出现即需整段重写**：

- `本研究` / `本文` / `论文`
- `作者` / `笔者`
- `认为` / `指出` / `提出`（主观判断动词）

替代方式：改用第三人称客观陈述。

| 禁用写法 | 合规替换 |
|---|---|
| 本研究构建了……框架 | 研究构建了……框架 |
| 本文认为学科建制的关键在于…… | 学科建制的关键在于…… |
| 作者指出，中国路径呈现…… | 中国路径呈现…… |

### J.2.2 结构要素

多数 C 刊摘要要求覆盖四要素，但表述不得叠床架屋：

```
问题（研究了什么）→ 方法（怎么研究的）→ 结果（发现了什么）→ 结论/意义（有何价值）
```

结果与结论不可混杂为同一段。

### J.2.3 字数

- 查清该刊字数区间（常见为 200—300 字）
- 超出须压缩，不足须补充；不要在摘要里填充废话凑字数

### J.2.4 摘要末句的政策口号化禁忌

摘要末句最易出现"政策报告腔"，须主动剔除。以下表述模式一经出现即需重写：

| 禁用模式 | 真实案例 | 替代方式 |
|---|---|---|
| 为 X 提供新动能 / 新支撑 / 新方向 / 新路径 | "为新型科技智库建设提供新动能新支撑" | 用判断句陈述结论的具体内涵 |
| 对 X 具有重要意义 / 深远影响 | "对智库治理转型具有深远影响" | 删除，或改为"研究框架在 X 范围内适用" |
| 推动 X 高质量发展 / 全面升级 | "推动智库治理体系高质量发展" | 改为对具体机制效果的判断 |
| 助力 / 赋能 / 引领 X | "助力科技决策咨询能力提升" | 删除；这类动词为口号化重灾区 |

**合规摘要末句应做到**：用一句判断式陈述说明结论的边界、适用范围或不确定性。例如：

> 研究框架适用于以政策外溢性为核心特征的咨询类知识生产；对市场化研究机构与商业智库的适用性尚待进一步验证。

## J.3 关键词

- 数量：通常 3—5 个，查期刊规定
- 选词原则：尽量使用《汉语主题词表》中已有的规范词；自创概念可保留 1 个，但需在正文中有明确定义
- 排列：从宽泛到具体，或按论文论点重要性排列

## J.4 正文引文格式

**数字编号制（GB/T 7714）与作者—年份制（APA）绝对不可混用。**

| 格式制 | 正文引文样式 | 适用期刊（举例） |
|---|---|---|
| 数字编号 | [1]、[2-4]、[5,7] | 《高等工程教育研究》《中国高教研究》 |
| 作者—年份 | （Smith, 2020）、（Smith & Jones, 2019） | 《华东师范大学学报（教育科学版）》 |

切换格式后需全文扫描，确保残留为零：
```bash
# 检查数字引用残留
grep -nE '\[[0-9]+-?[0-9]*\]' <file>
# 检查作者年份残留
grep -nE '\([A-Z][a-z]+.*[0-9]{4}\)' <file>
```

## J.5 参考文献

### J.5.1 EB/OL 条目的日期

GB/T 7714-2015 要求标注**发布日期**（或更新日期）+ **引用日期**，两者均需标注。

- 发布日期不可确认时：标注 `[不详]`，绝不编造估算日期
- 引用日期：写实际查阅的日期
- 格式：`[发布日期/更新日期][引用日期]`

```
错误：PURDUE UNIVERSITY. School of Engineering Education[EB/OL]. (2024-01-01)[2024-08-01].
正确：PURDUE UNIVERSITY. School of Engineering Education[EB/OL]. [不详][2024-08-01].
```

### J.5.2 外文作者姓名格式（GB/T 7714）

- 姓全大写，名缩写加点：`SMITH J`, `JONES A B`
- 三人及以上：列前三人，加 `et al.`（中文文献用"等"）
- 中国学者英文文献：姓在前全大写，名缩写，与期刊实际发表格式一致

### J.5.3 中文期刊文献

- 必须有起止页码；"1-13"需核实是否完整
- 卷号期号格式：`年份，卷（期）：起止页`

### J.5.4 文献数量与结构

C 刊投稿的参考文献常见问题：
- 中文文献过少（目标刊为中文刊，中文文献不足 30% 会被追问）
- EB/OL 网页来源过多（学术权重低，控制在总量 20% 以内）
- 缺少近 3 年的最新文献（显示对领域动态追踪不足）

### J.5.5 孤儿引用与僵尸文献扫描

多轮修订中最易出现的硬伤：参考文献表里有第 [n] 条，正文却没引用（**僵尸文献**）；或者正文出现 [m]，参考文献表里却找不到（**孤儿引用**）。两类问题被审稿人发现都会被打回。

扫描命令：

```bash
# 提取正文所有引用编号（不含参考文献表段）
sed '/^参考文献/,$d' <file> | grep -oE '\[[0-9]+\]' | sort -u > /tmp/cited.txt

# 提取参考文献表所有编号
sed -n '/^参考文献/,$p' <file> | grep -oE '^\[[0-9]+\]' | sort -u > /tmp/listed.txt

# 比对
diff /tmp/cited.txt /tmp/listed.txt
# 仅出现在 cited.txt 中：孤儿引用（正文引了但表里没有）
# 仅出现在 listed.txt 中：僵尸文献（表里列了但正文没引）
```

**两类问题的高发场景**：
- 多轮删段：删段时只删了正文，未连带删除对应参考文献条目 → 僵尸文献
- 重编号失败：调整引用顺序后重编号未覆盖全文 → 编号错位
- 替换文献：替换某条参考文献后，正文引用编号未同步更新 → 编号指向错条目

### J.5.6 高风险文献条目的强制核实清单

下列文献类型在投稿前须 **逐条二次核实**，不可凭印象保留。本清单整理自多个项目中真实出现的伪造或错误条目：

| 文献类型 | 核实要点 | 项目真实教训 |
|---|---|---|
| arXiv 论文 | 验证编号格式 YYMM.nnnnn（年月+4 至 5 位序号）；2025 年起 arXiv 启用 5 位序号，旧 4 位格式已不够用；用编号直接访问 arxiv.org 核对作者与题名 | 曾出现"arXiv 2602.24176"，作者归属错误 |
| 标注为近 1—2 年的中文学术专著 | CNKI 图书数据库 + 出版社官网交叉核实；尤其当作者是该领域知名学者时——知名学者的最新著作最易被记忆错位 | 曾出现"潘教峰 2025 年专著"，无法在任何数据库验证 |
| 国外智库报告与学术论文的卷期页码 | 核对官方网站原文；CEBRI、Brookings、CSET 等的卷期编号易出错 | 曾出现 CEBRI 文章年份、卷、期、页码四项均有误 |
| EB/OL 类网页 | 链接失效率高；标注引用日期外，必要时加注"如链接失效，可在 X 数据库检索关键词 Y" | 多次出现链接失效 |
| 作者自引文献 | 即使是作者本人的论文，卷期页码也常记错；通过 CNKI 重新核对 | — |
| 中外文姓名拼写一致的不同作者 | 验证是否为同一人；中文学者英文论文署名常与中文一致性核查 | — |

**核实失败的处理原则**：

- 30 分钟内无法在权威数据库找到的条目 → 删除，不要"待查后投稿"
- 找到替代条目的 → 替换并同步更新正文引用
- 必须保留但卷期不全的 → 标注"可能不准确，建议作者核实"，并在投稿前由作者本人完成核对

## J.6 匿名化（双向匿名审稿）

若目标期刊采用双向匿名审稿：

- 正文及首页：只保留题目、摘要、关键词；**不出现**作者姓名、单位、基金项目
- 作者信息另起一页置于全文末尾，内容包括：姓名、单位、职务/职称、研究方向、联系地址、邮编、电话、电子邮件
- 英文摘要与英文题目：一并置于末页
- 自引文献处理：正文中的自引须匿名化，如"参见[作者]（2020）"→ 改为"参见[匿名]（2020）"或删去

---

## J.7 英文摘要与英文关键词

- 英文题目：主要实词首字母大写（title case）
- 英文摘要：与中文摘要信息对应，但不必逐句直译；时态用一般现在时为主
- 英文关键词：与中文关键词一一对应，使用领域通用术语而非字面直译

## J.8 投稿前最终扫描命令

```bash
# 摘要禁用词
grep -nE '本研究|本文|作者|笔者|认为|指出' <摘要段>

# 摘要末句口号化
grep -nE '新动能|新支撑|新方向|新路径|高质量发展|助力|赋能|引领' <摘要段>

# 正文"本研究/本文"频次
grep -cE '本研究|本文' <file>

# 硬禁用词（风格底盘）
grep -nE '深度[^度学]|显著[^性]|赋能|恰恰|重塑' <file>

# 句首禁语
grep -nE '值得注意|不难发现|显而易见|综上所述|由此可见' <file>

# 破折号插入语
grep -nE '[^—]——[^—]' <file>

# 引文格式残留检查（切换格式后用）
grep -nE '\[[0-9]+-?[0-9]*\]' <file>

# 孤儿引用与僵尸文献比对（详见 J.5.5）
sed '/^参考文献/,$d' <file> | grep -oE '\[[0-9]+\]' | sort -u > /tmp/cited.txt
sed -n '/^参考文献/,$p' <file> | grep -oE '^\[[0-9]+\]' | sort -u > /tmp/listed.txt
diff /tmp/cited.txt /tmp/listed.txt
```

## J.9 投稿系统事项（Claude 不操作，提示作者自行完成）

以下事项须作者在投稿系统中独立完成，Claude 不代为填写：

1. 投稿系统注册与登录
2. 作者个人信息填写（身份证号、单位、职务、联系方式）
3. 基金项目信息（如实填写，无则删除该行）
4. 推荐/回避审稿人（如期刊有此选项）
5. 版权转让协议签署
