# 07. 投稿适配

定稿前的最后一里。把"一篇好论文"变成"一篇符合目标期刊体例的可投论文"。

---

## 一、投稿前的硬清单

每篇投稿前必检：

```
[ ] 中文标题 ≤ 20 字（主标题），副标题 ≤ 25 字
[ ] 中文摘要符合期刊字数要求（《中国高教研究》≤ 288 字）
[ ] 中文关键词数量符合（多数 3-5 个或 3-8 个）
[ ] 英文题目（与中文对应）
[ ] 英文摘要 250-350 词
[ ] 英文关键词（与中文对应）
[ ] 作者信息完整（姓名、单位、职务、职称）
[ ] 通讯作者标注（多作者时）
[ ] 基金标注（如有）
[ ] 参考文献按 GB/T 7714-2015，按首次出现顺序编号
[ ] 参考文献无缺号、无引而未列、无列而未引
[ ] 全文图表清晰，编号正确
[ ] 涉台 / 政治措辞合规
[ ] 禁用词扫描通过
[ ] 元叙述密度 ≤ 5 处
[ ] 结语含局限说明
[ ] 字数符合期刊要求
[ ] 投稿系统所需的额外文件（投稿信、author bio、原创性声明等）
```

## 二、主要 C 刊体例速查

### 2.1 《中国高教研究》

| 项 | 要求 |
|---|---|
| 题目 | 主标题 + 副标题（用"——"连接）|
| 摘要 | ≤ 300 字（建议 ≤ 288）|
| 关键词 | 3-8 个，分号分隔 |
| 正文字数 | 通常 13000-15000 字 |
| 结构 | 不设独立"结语"节 |
| 注释 | 不用脚注，特殊说明嵌入正文括号或合并到参考文献 |
| 作者简介 | 首页脚注，含姓名（出生年）、性别、单位（详细到二级单位）、职务、职称 |
| 基金 | 首页脚注，"基金项目：XXXX（编号）" |
| 参考文献 | GB/T 7714-2015，行内编号 [1][2] |
| 图表 | 黑白为主，正文插图风格朴素 |

### 2.2 《高等工程教育研究》

| 项 | 要求 |
|---|---|
| 题目 | 简洁，可不带副标题 |
| 摘要 | ≤ 300 字 |
| 关键词 | 3-8 个 |
| 正文字数 | 通常 10000-15000 字 |
| 结构 | 可设结语节 |
| 注释 | 可有少量脚注 |
| 参考文献 | GB/T 7714-2015 |

### 2.3 《教育研究》

| 项 | 要求 |
|---|---|
| 题目 | 一般 ≤ 25 字 |
| 摘要 | ≤ 300 字 |
| 关键词 | 3-8 个 |
| 正文字数 | 通常 15000-20000 字 |
| 注释 | 可有脚注 |
| 风格 | 偏向理论性、思辨性 |

### 2.4 《管理世界》

| 项 | 要求 |
|---|---|
| 体例严格 | 投稿前必须仔细阅读最新投稿指南 |
| 字数 | 通常 18000-25000 字 |
| 偏好 | 量化研究、严谨方法、明确贡献 |

### 2.5 通用 C 刊体例参照

如目标期刊未在上述列表，参考以下通用规则：

- 摘要 ≤ 300 字（多数）
- 关键词 3-5 个或 3-8 个
- 参考文献 GB/T 7714-2015
- 正文字数 12000-20000 字

无论哪本刊，**投稿前必须读最新投稿指南**——期刊体例每年可能微调。

## 三、首页脚注的处理

中国 C 刊（特别是《中国高教研究》）惯用首页脚注（页脚注释）放：

- 基金项目
- 作者简介
- 通讯作者标注

格式示例：

```
基金项目：[基金全称]（[项目编号]）。
作者简介：符杰（1992—），男，清华大学教育学院/联合国教科文组织
国际工程教育中心，助理研究员；乔伟峰（通讯作者），男，清华大学
教育学院党委副书记/联合国教科文组织国际工程教育中心，副研究员。
```

注意：

- 出生年用全角破折号
- 单位详细到二级单位
- 多个作者用分号分隔
- 通讯作者明确标注

如目标期刊不设首页脚注（如部分刊用文末作者简介），按其体例调整。

## 四、英文摘要写作

### 4.1 结构

```
[第 1 句：研究问题，1 句]
[第 2-3 句：方法概述，1-2 句]
[第 4-7 句：核心发现，3-4 句]
[第 8-9 句：贡献或政策含义，1-2 句]
```

### 4.2 写作要点

- 主动语态："We compare..." 优先于 "A comparison is conducted..."
- 句子短，避免长定语
- 关键概念附中文括号：strategic restructuring (战略性重构)
- 用专业术语：legislative conditionality, policy mix, skill formation institutions

### 4.3 示例

```
Strategic restructuring of integrated circuit (IC) talent cultivation systems
has shifted from an internal higher-education issue to a strategic concern
linked to industrial capacity and national security. Following an analytical
chain of "policy instrument mix—organizational and governance structures—
cultivation and capability-formation mechanisms", this paper conducts a
structured focused comparison of four principal cases—the United States,
Japan, South Korea, and China's Taiwan region—with the European Union as
a contrast case. We find that the four principal cases all incorporate
talent cultivation into industrial security frameworks, but diverge into
four policy paths: legislative conditionality, administrative mobilization,
fiscal concentration, and special legislative authorization. The EU,
constrained by dispersed educational sovereignty, presents a fifth path
of supranational coordination. Industrial structure and value-chain
position, policy instrument mix and execution capacity, and university
governance and skill formation institutions constitute the institutional
basis for path divergence. For China, the policy linkage between industrial
incentives and talent cultivation, tiered curricula stratified by technology
trajectories, and localized institutional pilots offer conditionally
transferable lessons; exclusive contractual cultivation and defense-coordinated
models, however, are constrained by specific institutional conditions and
have limited room for transplantation.

Keywords: integrated circuit; semiconductor; strategic restructuring;
engineering education; talent cultivation
```

约 280 词。

## 五、致谢与利益冲突声明

部分 C 刊要求致谢或利益冲突声明。

### 5.1 致谢

```
本文初稿曾在 X 会议宣讲，感谢 Y 学者的评议；感谢 Z 学者的反复审读
与建设性意见。文责自负。
```

注意：

- 不要致谢"前辈学者"等空话
- 致谢的人应该在论文写作中有具体贡献
- "文责自负"是常见尾句

### 5.2 利益冲突声明

如有要求：

```
作者声明无利益冲突。
```

## 六、投稿系统准备

不同期刊投稿系统略有差异，但通常需要：

1. 投稿信（Cover Letter）—— 简述论文主题、贡献、建议审稿人
2. 论文正文（含摘要、关键词、正文、参考文献）
3. 作者信息表
4. 原创性声明
5. 版权转让声明（部分期刊）
6. 推荐 / 回避审稿人名单（部分期刊）

### 6.1 投稿信模板

```
尊敬的[期刊名]编辑部：

兹投稿《[论文题目]》一文，敬请审阅。

本文聚焦于[一句话研究问题]，采用[一句话方法概述]，发现[一句话核心发现]。
本文的学术贡献在于[一句话贡献]。

本文未在其他刊物发表，亦未同时投稿其他期刊。如蒙采用，作者将按贵刊
要求作进一步修改。

作者：[姓名]
单位：[单位全称]
联系方式：[邮箱、电话]
日期：[日期]
```

### 6.2 推荐审稿人

如需推荐：

- 推荐 3-5 位本研究领域的资深学者
- 不推荐与作者有合作关系的学者
- 推荐时附学者的研究方向简介

### 6.3 回避审稿人

如有充分理由（如学术争议、利益冲突），可申请回避特定审稿人。需说明回避理由。

## 七、投稿后的预期

C 刊投稿后的常见时间线：

```
Day 0：投稿
Day 7-30：初审（编辑判断是否送外审）
Day 30-90：外审（通常 2-3 位匿名审稿人）
Day 90-180：审稿意见返回（小修 / 大修 / 退稿）
Day 180-300：修改后再次审稿（如适用）
Day 300-450：录用通知
Day 450-600：刊出
```

从投稿到刊出，6 个月到 18 个月是正常区间。

---

## 八、投稿前的最后一遍通读

投稿前必须做 **一次完整的纸面通读**（打印或离屏阅读）。这是抓最后一类错误的最有效方法：

- 错字、漏字
- 标点格式（半角/全角混用）
- 引用编号错位
- 表格数字错误
- 图表与正文口径不一致
- 章节序号跳跃

屏幕阅读 6-8 小时后大脑会自动跳过明显错误，纸面阅读能重新激活注意力。

## 九、被退稿后的应对

C 刊退稿率通常 70%-90%，被退稿是常态。

### 9.1 第一步：判断退稿类型

- **编辑直接退稿**（未送外审）：选题不符 / 体例不符 / 质量明显不达标
- **外审后退稿**（含审稿意见）：质量未达录用线 / 创新性不足 / 方法问题

### 9.2 第二步：消化审稿意见

即使被退稿，审稿意见也是宝贵反馈。按 P0/P1/P2 整理（见 `06-revision-loop.md`）。

### 9.3 第三步：决定下一步

| 情况 | 行动 |
|---|---|
| 致命问题可修复 | 大改后投同档期刊 |
| 创新性不足 | 重新评估选题或重大重构 |
| 方法争议 | 评估是否需改变方法 |
| 选题不符该刊 | 改投选题更匹配的期刊 |

### 9.4 第四步：不要赌气

被退稿后立即投其他刊容易出错。建议冷静 1-2 周再修改重投。

## 十、投稿伦理

### 10.1 不能做的

- 一稿多投（同时投多本期刊）
- 抄袭（包括自我抄袭）
- 编造数据 / 文献
- 不当署名（搭车作者、漏列贡献者）

### 10.2 应该做的

- 单本期刊有结果后再考虑改投
- 所有引用注明出处
- 作者署名按贡献排序，标注通讯作者
- 主动声明利益冲突

## 十一、本节常用产出物

投稿前应该有一个 "submission package" 文件夹，含：

```
submission/
├── manuscript-final.docx       # 正文最终版
├── cover-letter.docx           # 投稿信
├── english-abstract.docx       # 英文摘要（独立文件）
├── author-info.docx            # 作者信息
├── figures/                    # 图表（PNG + PDF）
│   ├── fig1_framework.png
│   ├── fig1_framework.pdf
│   └── ...
├── declaration-originality.docx  # 原创性声明
└── submission-checklist.md     # 本节硬清单的勾选记录
```

打包好后再投稿。投稿过程中临时找文件容易出错。
