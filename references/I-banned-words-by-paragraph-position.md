# I. 段落位置敏感词

某些词在特定位置（段首、段尾、章节起始、摘要等）出现时风险显著高于其他位置。本文件给出按位置分组的敏感词清单和处理方案。

---

## 一、段首禁语（绝对禁用）

以下短语**绝不**作为段落起首：

```
值得注意的是
值得一提的是
值得关注的是
不难发现
不难看出
显而易见
显然
毋庸置疑
不可否认
众所周知
一言以蔽之
总而言之
总的来说
综上所述
综合以上分析
综合上述安排
综合而言
归根结底
归结而言
由此可见
不言而喻
可以这样说
有理由相信
从某种意义上讲
某种程度上
在一定意义上
从这个角度看
从这个意义上说
```

替换原则：**直接给出判断句**，不需要任何引导语。

---

## 二、段尾禁语（强烈避免）

段尾出现的"升华句"是 AI 检测最敏感的特征之一。以下模式应避免：

```
这意味着 X 正在转向 Y
这一变化反映了 X 的转变
这一现象表明 X 的趋势
上述变化共同指向一种 X 转型
三者共同构成了 X 的制度基础
该案例展示了 X 的可能性
本研究的发现支持 X 的观点
```

特征：

- 用"这意味着""这一……""共同……"等开头
- 把段落内容做一个范式性总结
- 不增加新信息，只重复主旨

**正确的段尾**：

```
- 一个具体判断（不是抽象总结）
- 一个限定或边界（不是无条件断言）
- 一个反例或例外（如适用）
- 直接结束，不需要"收口"
```

---

## 三、章节起始禁语

每章 / 每节开头应避免：

```
本节将首先 X 其次 Y 最后 Z
本节首先 X 然后 Y
为了 X 本节将 Y
接下来本研究将
下面将分别讨论 X、Y、Z
```

特征：

- 元叙述（告诉读者本节要做什么）
- 顺序铺陈（首先 / 其次 / 最后）
- 与节标题重复

**正确的章节起始**：

```
- 一个具体的判断或事实，直接进入实质
- 上下文的衔接句（如承接上一节）
- 本节核心论点的明确陈述
```

例：

```
错误：
> 本节将首先分析五个案例的政策工具类型，其次讨论其内在一致性，
> 最后总结政策维度的差异。

正确：
> 五个案例在政策工具选择上呈现出明显的路径分化。美国选择了法案约束……
```

---

## 四、章节小结禁语

每章末尾的小结段应避免：

```
综上所述
综合以上分析
通过上述讨论
通过本节的分析
本节的讨论表明
本节的分析显示
小结一下
总结而言
归纳起来
```

**正确的小结**：

```
- 不写"小结"——让段落自然结束
- 如必须有承上启下，写一句具体判断 + 引出下章
- 不重复本章主旨
```

例：

```
错误：
> 综合以上分析，政策维度的差异直接决定了组织维度的结构选择。

正确：
> 政策工具的差异并不机械决定组织选择——韩国的案例显示，行政动员
> 可能在缺少法案约束的条件下达到相近效果。组织维度的进一步差异
> 将在下一节展开。
```

## 五、摘要敏感词

摘要（≤ 288 字）字字珍贵，更要避免空话。

### 5.1 摘要禁语

```
本文（开头）—— 摘要直接给结论更有力
本研究（开头）—— 同上
为了 X 本文采用 Y 方法 —— 信息密度低
通过对 X 的 Y 分析 —— 句式 AI 化
得出以下结论 —— 信息为零
具有重要意义 —— 套话
为 X 提供借鉴 —— 套话
研究表明 —— 应改为直接陈述
```

### 5.2 摘要应有的元素

```
1. 一句研究问题
2. 一句方法概述
3. 三句核心发现（具体）
4. 一句政策含义或贡献
```

### 5.3 摘要示例

```
错误：
> 本文针对集成电路人才培养这一重要议题，采用比较研究方法，对五个
> 发达经济体进行了深入分析。研究表明，发达经济体在人才培养方面
> 具有显著的差异。本文得出以下结论：政策、组织、培养三个维度
> 存在密切联系。本研究为我国相关政策制定提供了有益借鉴。

正确：
> 集成电路人才培养已从高校内部议题上升为关乎产业能力与国家安全的
> 战略问题。沿"政策工具组合—组织与治理结构—培养与能力生成机制"
> 的分析链条，本文对四个主案例与一个对照案例进行结构化聚焦比较。
> 四个主案例均将人才培养纳入产业安全框架，并在政策工具上分化为
> 法案约束、行政动员、财政集中与专项立法授权四种路径。产业结构与
> 价值链位置、政策工具组合与执行能力、高校治理与技能形成制度三组
> 变量，构成路径分化的制度基础。
```

## 六、引言敏感词

引言尤其是第一段，最容易出现 AI 痕迹。

### 6.1 引言禁开头

```
在 X 背景下                  —— AI 偏好
随着 X 的发展                —— 套话
近年来 X 受到广泛关注          —— 信息低密度
X 是当前的重要议题            —— 套话
X 作为 Y 的重要组成部分        —— 套话
```

### 6.2 引言推荐开头

- 一个具体数据（"2024 年全球半导体市场规模达 5240 亿美元"）
- 一个具体事件（"2022 年 8 月，美国总统签署 CHIPS Act"）
- 一个直接论断（"集成电路人才培养已不再是单纯的高校议题"）

---

## 七、结语敏感词

结语 300-500 字，每一句都要精挑。

### 7.1 结语禁语

```
本文系统研究了 X —— 信息为零
本文得出以下结论 —— 套话
本研究的主要贡献在于 —— 可保留但要具体
未来研究方向包括 —— 套话
有待进一步研究 —— 空话
将持续关注 —— 套话
```

### 7.2 结语必含元素

```
1. 一句研究问题回顾
2. 三句核心发现
3. 一句贡献声明
4. 两条具体研究局限
5. 一句下一步可能的研究方向（具体的，不是套话）
```

## 八、案例分析节敏感词

案例并列段最容易出现"五个案例完美对称"的 AI 痕迹。

### 8.1 案例段禁开头模板

```
X 通过 Y 实现 Z
X 在 Y 方面采取 Z 的做法
X 的特征是 Y
X 政府推动了 Y
```

如果五个案例都用同样模板，重写。

### 8.2 案例段推荐变化

```
案例 1：[直接事实]。美国 2022 年通过 CHIPS Act……
案例 2：[对比起句]。日本的做法不同。它将……
案例 3：[特征起句]。韩国的特征是行政动员的高强度：尹锡悦政府……
案例 4：[时间起句]。2021 年，台湾地区当局通过……
案例 5：[结构起句]。欧盟受困于教育主权分散，被迫……
```

---

## 九、政策启示节敏感词

政策启示节最容易出现"应该……必须……"的政策文件话语。

### 9.1 启示节禁语

```
应该 X       —— 单向倡议
必须 X       —— 单向倡议
应当 X       —— 单向倡议
亟需 X       —— 单向倡议
应进一步 X   —— 套话
应不断 X     —— 套话
为此应当 X   —— 套话
建议 X       —— 太空泛
亟待加强 X   —— 政策文件话语
统筹推进 X   —— 政策文件话语
扎实推进 X   —— 政策文件话语
着力解决 X   —— 政策文件话语
```

### 9.2 启示节推荐写法

按"借鉴方向—制度前提—适配性判断"三段式：

```
启示 X：[借鉴方向]

[借鉴方向，1-2 句具体说做什么]

[制度前提，2-3 句：该做法在原案例中依赖什么制度条件]

[适配性判断，2-3 句：我国是否具备相应条件，需做什么调整]
```

## 十、扫描脚本

```bash
#!/bin/bash
# 按位置扫描敏感词

FILE=$1

echo "=== 段首禁语 ==="
grep -nE '^[。\s]*(值得注意|不难发现|显而易见|综上所述|综合以上|总而言之|由此可见|不言而喻|归根结底|有理由相信)' "$FILE"

echo ""
echo "=== 段尾'升华句'候选 ==="
grep -nE '(这意味着|这一变化|这一现象|上述变化|三者共同).{0,30}(转向|表明|意味|构成|共同)' "$FILE"

echo ""
echo "=== 章节起始元叙述 ==="
grep -nE '本节将首先|本节首先.{0,20}其次|为了.{0,20}本节将' "$FILE"

echo ""
echo "=== 章节小结禁语 ==="
grep -nE '通过上述讨论|通过本节的分析|本节的讨论表明|本节的分析显示|总结而言' "$FILE"

echo ""
echo "=== 摘要 / 引言禁开头 ==="
grep -nE '^本文针对|^本研究针对|在.{0,15}背景下' "$FILE"

echo ""
echo "=== 启示节单向倡议 ==="
grep -nE '^应该|^必须|^应当|^亟需|^应进一步|^应不断|^亟待|^建议' "$FILE"

echo ""
echo "=== 政策文件话语 ==="
grep -nE '统筹推进|扎实推进|着力解决|聚焦发力|精准发力|靶向发力|高质量发展|深化改革' "$FILE"
```

## 十一、修订时的位置感

修订时按位置敏感度排序：

```
最高优先级：
- 结语段（每一句都要精挑）
- 摘要（字字珍贵）
- 引言第一段（决定审稿人第一印象）

高优先级：
- 政策启示节（套话重灾区）
- 比较与解释节的总结句
- 各章节末尾小结句

中等优先级：
- 各章节起始句
- 案例并列段的开头模板
- 段尾"升华句"

一般优先级：
- 正文中部段落
- 方法节
- 文献综述节
```

按这个顺序修，效率最高。
