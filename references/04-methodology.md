# 04. 方法与研究设计

C 刊审稿对方法的严苛程度仅次于理论。方法节是审稿人验证"作者会做研究"的关键证据。

---

## 一、方法节的标准结构

方法节通常 1500-2500 字，结构如下：

```
## X. 研究方法

### X.1 研究设计 / 方法选择
[300-500 字：本研究采用什么方法，为什么]

### X.2 案例 / 样本 / 数据
[400-700 字：研究对象的选择、来源、标准]

### X.3 操作化 / 测量
[400-700 字：核心概念如何观察、编码、测量]

### X.4 分析步骤
[300-500 字：怎么从数据走到结论]

### X.5 信效度 / 可复核性 / 局限
[200-400 字：方法的边界与可靠性]
```

X.3 是高频被审稿人抓的环节。核心概念没操作化，前面写得再好都会被打回。

---

## 二、核心概念的操作化

### 2.1 什么是操作化

操作化（operationalization）= 把抽象概念转化为可观察、可比较的指标或维度。

例：
- 抽象概念："战略性重构"
- 操作化：四个可观察维度——治理权重再分配、资源配置与平台化、培养逻辑技术路线化、国际嵌入与流动

### 2.2 操作化的检查标准

| 检查 | 通过标准 |
|---|---|
| 维度是否清晰 | 每个维度能用一句话定义 |
| 维度之间是否互斥 | 任一现象只能归入一个维度 |
| 维度是否穷尽 | 概念的所有方面都被维度覆盖 |
| 是否可观察 | 每个维度有可识别的经验对应物 |
| 是否可编码 | 每个维度有明确的等级或类型 |

### 2.3 操作化矩阵的呈现

把操作化的结果用表格呈现：

```
| 维度 | 定义 | 编码 0 | 编码 1 | 编码 2 |
|---|---|---|---|---|
| 治理权重再分配 | ... | 无变化 | 部分调整 | 全面再分配 |
| 资源配置与平台化 | ... | ... | ... | ... |
```

编码标准必须可复核——另一位研究者按同样标准能得出相近结论。

## 三、比较研究的方法学

### 3.1 比较研究的常见类型

| 类型 | 代表方法 | 适用 |
|---|---|---|
| 最相似系统设计 | Most-Similar Systems Design | 在相似案例中找差异原因 |
| 最相异系统设计 | Most-Different Systems Design | 在差异案例中找共同因素 |
| 结构化聚焦比较 | Structured Focused Comparison (George & Bennett 2005) | 多案例系统比较 |
| 定性比较分析 | QCA (Ragin) | 条件组合分析 |
| 过程追踪比较 | Comparative Process Tracing | 因果机制比较 |

### 3.2 结构化聚焦比较的关键

这是 C 刊比较研究最常用的方法。核心要素：

1. **标准化提问**：对每个案例提同样的问题
2. **聚焦**：只问与研究问题直接相关的问题
3. **可比性**：所有案例在相同维度上被刻画

标准化提问示例：

```
对每个案例提以下问题：
Q1：该案例采用了哪些政策工具？
Q2：政策工具之间的一致性如何？
Q3：政策工具如何转化为组织安排？
Q4：组织安排如何支撑培养机制？
Q5：本案例与其他案例的差异在哪？
```

### 3.3 案例选择的论证

C 刊审稿人会问：为什么是这几个案例而不是别的？必须给出明确论证：

| 选择标准 | 说明 |
|---|---|
| 代表性 | 这些案例代表了主要类型 / 主要路径 / 主要规模 |
| 重要性 | 这些案例本身在领域内有重要地位 |
| 可比性 | 这些案例在关键维度上可比 |
| 差异性 | 这些案例在关键维度上有差异，便于解释 |
| 数据可得性 | 这些案例有充分的资料支撑 |

四个以上理由能站住，案例选择就过关。

### 3.4 对照案例的处理

如果有"对照案例"（与主案例不可比但提供参照），必须在方法节明确说明：

> 本研究选取 X、Y、Z 三个主案例进行结构化聚焦比较，并选取 W 作为对照案例。
> W 因其特殊性（如：超国家治理结构）不与主案例直接比较，但其制度安排可为
> 主案例的解释提供反向参照。

## 四、数据 / 资料来源

### 4.1 三类常见资料

| 类型 | 示例 | 可靠性 |
|---|---|---|
| 一手政策文本 | 法律、规划、白皮书 | 高 |
| 二手学术研究 | 期刊论文、专著 | 高 |
| 行业报告 | 协会报告、咨询公司报告 | 中 |
| 新闻报道 | 主流媒体 | 中低 |
| 灰色文献 | 政府网站发布、机构网页 | 低-中 |

### 4.2 资料的口径说明

一篇严谨的比较研究，所有数据必须说明口径：

- 时间范围：从 X 年到 Y 年
- 统计口径：包括 / 不包括什么
- 来源机构：哪个机构发布
- 引用方式：原数据 / 二次加工

### 4.3 政策文本的处理

引用政策文本时：
- 标注文件全称
- 标注发布机构
- 标注发布年份
- 标注引用的具体条款或段落（如适用）

例：
> 美国《2022 年芯片与科学法案》（CHIPS and Science Act of 2022）第 X 节
> 规定，获得 NOFO 资助的企业须承诺培养 Y 名半导体工程师。

## 五、方法节的"局限"段

每篇 C 刊论文必须主动说明研究方法的局限。审稿人喜欢看到作者有自觉性。

### 5.1 局限的常见类型

| 局限 | 表达模板 |
|---|---|
| 案例覆盖局限 | "本研究的发现基于 X 个案例，对其他类型案例的适用性需进一步检验" |
| 数据局限 | "本研究主要依赖一手政策文本与二手学术研究，缺乏直接的运行数据" |
| 时间局限 | "本研究的观察时段截至 X 年，X 年后的演变需后续追踪" |
| 方法局限 | "本研究采用 X 方法，未能就 Y 提供因果推断" |
| 视角局限 | "本研究采用 X 视角，未能涵盖 Y 视角下的解释" |

### 5.2 局限的"度"

太弱：
> 本研究存在一定局限，需后续进一步深化。
（空话，没说什么）

太强：
> 本研究的方法存在严重问题，结论的可信度有限。
（自杀式表述，不要写）

恰当：
> 本研究的发现基于五个案例，对其他类型案例（如东南亚新兴半导体国家）
> 的适用性需进一步检验。本研究主要依赖政策文本和学术研究，缺乏对培养
> 效果的直接评估，这一评估有待后续基于产出数据的研究补足。
（具体、有限度、有指向）

## 六、量化研究的方法节

如果论文采用量化方法，方法节须包括：

| 要素 | 说明 |
|---|---|
| 数据来源 | 哪个数据库 / 哪个调查 / 自行收集 |
| 样本范围 | 时间范围、地理范围、对象范围 |
| 样本量 | N = X，剔除 Y 后剩 Z |
| 变量定义 | 因变量、自变量、控制变量 |
| 模型设定 | 模型方程或方法名 |
| 统计软件 | Stata / R / Python 等 |
| 稳健性检验 | 用什么方法检验稳健性 |

---

## 七、质性研究的方法节

如果采用田野、访谈、文本分析等方法，须包括：

| 要素 | 说明 |
|---|---|
| 研究场域 | 在哪做 |
| 进入方式 | 怎么获得访谈 / 田野准入 |
| 资料类型 | 访谈、观察、文档 |
| 资料量 | N 次访谈 / N 份文档 |
| 受访者特征 | 身份、背景（脱敏处理）|
| 分析方法 | 主题分析 / 扎根理论 / 话语分析等 |
| 伦理审查 | 如适用 |

---

## 八、方法节的写作禁忌

1. **不要用"科学方法"等空话**
2. **不要把"文献综述"当方法**——文献综述是文献综述，不是方法
3. **不要把"逻辑分析"当方法**——任何论文都用逻辑分析
4. **不要回避局限**——主动说局限比被审稿人挑出来好
5. **不要假装严谨**——做了什么写什么，没做的别说做了

---

## 九、与方法节配合的章节

方法节定下的方法必须在正文执行。常见的对应关系：

- 方法节说"结构化聚焦比较" → 案例分析章必须按统一提问框架
- 方法节说"四维编码" → 必须有编码矩阵表
- 方法节说"过程追踪" → 必须呈现关键节点的因果序列
- 方法节说"政策工具组合分析" → 必须有政策工具分类与组合判断

如果方法节承诺了某方法但正文没做，是 P0 级硬伤。

---

## 十、方法节的常见审稿质疑与回应

| 质疑 | 回应方式 |
|---|---|
| "为什么选这几个案例？" | 在 X.2 用 4-5 条理由说清 |
| "操作化标准是什么？" | 在 X.3 给出编码矩阵 |
| "怎么保证可复核性？" | 在 X.5 说明编码规则、原始材料可获取性 |
| "结论怎么从数据推出来？" | 在 X.4 列出分析步骤 |
| "方法和理论怎么对应？" | 在 X.1 末段明确说明 |
| "数据是否最新 / 全面？" | 在 X.5 承认局限并说明数据更新策略 |
