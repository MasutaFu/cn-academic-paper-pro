# F. 自评清单与多专家审稿模板

本文件给出标准化的自评流程，让另一个 Claude 实例能像一位资深审稿人那样审视稿件。

---

## 一、四专家透镜

每次自评必须切换以下四个视角，每个视角只看本视角关心的问题：

### 1.1 学科领域专家（半导体 / 教育 / 政策等，按论文主题切换）

关注：
- 事实是否准确（数据、年份、机构名、技术参数）
- 概念使用是否专业（避免外行表述）
- 案例选择是否有代表性
- 是否遗漏关键事实或近期重要发展
- 行业实践的细节是否符合实际

典型质问：
- "数据来源是否可靠？是否最新？"
- "这个案例真的支持这个论点吗？还是只是表面相似？"
- "是否遗漏了 X 这个更重要的案例？"
- "技术细节是否准确？"

### 1.2 方法论专家

关注：
- 研究问题是否清晰
- 方法是否与问题匹配
- 案例选择标准是否透明
- 操作化是否到位
- 比较框架是否一致
- 编码 / 测量是否可复核

典型质问：
- "为什么是这五个案例而不是别的？"
- "'战略性重构'怎么观察？标准是什么？"
- "比较的维度对所有案例都适用吗？"
- "结论可以从证据中推出来吗？"

### 1.3 理论专家

关注：
- 理论框架是否实质参与正文分析
- 理论概念是否使用准确
- 理论之间的关系是否清晰
- 是否与已有学术讨论形成对话
- 创新性贡献是否清晰

典型质问：
- "三螺旋理论在正文哪里真正用了？还是只引言挂旗？"
- "Etzkowitz 的'边界条件'是这个意思吗？"
- "和 X 学者的研究相比，本文贡献在哪？"
- "理论选择是否过度——四个理论是否都必要？"

### 1.4 期刊主编

关注：
- 选题是否符合期刊偏好
- 题目是否吸引人
- 摘要是否突出核心发现
- 字数是否符合要求
- 体例是否规范
- 政治措辞是否合规
- 是否有明显的 AI 化痕迹
- 阅读体验是否流畅

典型质问：
- "我会接收这篇稿子吗？"
- "题目能让目标读者有阅读冲动吗？"
- "摘要的前 50 字是否抓住了眼球？"
- "这能上得了我们刊吗？"

## 二、一票否决项清单

以下任何一项不过，都不能投稿。每次自评必跑：

| 检查项 | 通过标准 | 检查方法 |
|---|---|---|
| 核心概念是否操作化 | 有明确的观测维度和编码标准 | 找到对应段落，看是否有可操作的描述 |
| 方法是否可复核 | 案例选择、数据来源、编码规则均透明 | 找方法节，看三要素是否齐全 |
| 理论是否实质参与 | 每个引入的理论在正文有具体调用 | 全文 grep 理论关键词，看分布 |
| 因果 / 分析链条是否贯通 | 引言承诺的框架在每章实质执行 | 逐章对照框架 |
| 证据链是否可追溯 | 每个数据 / 事实都有来源 | 抽查若干数据点 |
| 涉台表述是否合规 | 全文"中国台湾地区"，无独立"台湾"作为国家 | grep 涉台关键词 |
| 政治措辞无风险 | 无敏感词汇、无可能引发问题的表述 | 通读高风险段 |
| 创新性是否可辨识 | 结语有清晰的贡献声明 | 看结语 |
| 摘要与正文一致 | 摘要承诺的发现在正文充分论证 | 摘要逐句对应正文 |
| 参考文献体系完整 | 无缺号、无引而未列、按首次出现顺序 | 跑参考文献检查脚本 |
| 英文摘要 | 已撰写，250-350 词 | 看是否存在 |
| 字数符合期刊要求 | 中文摘要 ≤ 期刊上限，正文在常见 13000-18000 | wc 统计 |

## 三、AI 痕迹检测（结构层）

除 H-banned-words.md 的词汇扫描外，需检查以下结构层特征：

### 3.1 段落对称性扫描

```bash
# 检查并列段是否同构（五段开头是否完全平行）
awk '/^一、|^二、|^三、|^四、|^五、/ {print NR": "$0}' <file>
# 看每个一级标题下的小节开头句式
```

判断：如果案例并列段每段都以"X 通过 Y 实现 Z"开头，重写。

### 3.2 元叙述密度

```bash
grep -cE '本文|本节|下文将|上文已|如前所述' <file>
# 应 ≤ 5，集中在引言和结语
```

### 3.3 段尾升华句

人工扫描每段最后一句：是否在重复段落主旨？是否在做"意味着……正在转向……"式收口？是则改写。

### 3.4 单向论证

每段是否包含至少一处转折或限定？没有就在合理位置补一处。

### 3.5 章节小结的规范性

每章末尾是否有"综上所述""综合以上分析"开头的小结段？有就删除或改写为带具体判断的承上启下句。

## 四、终审评分模板

输出格式：

```markdown
## 终审评分表

| 维度 | 评分 | 核心判断 |
|---|---|---|
| 选题与问题意识 | ★★★★★ | 选题重大、时效性强 |
| 理论框架 | ★★★★☆ | 多理论整合实质参与 |
| 研究设计与方法 | ★★★★☆ | 框架完整、编码透明 |
| 经验材料与证据 | ★★★★☆ | 材料密度高、口径清晰 |
| 分析深度 | ★★★★☆ | 因果传导差异化 |
| 文献对话 | ★★★☆☆ | 中文期刊文献仍偏少 |
| 写作质量 | ★★★★☆ | AI 痕迹基本消除 |
| 规范与格式 | ★★★★☆ | GB/T 7714 基本规范 |

**总评**：A- / 修改后可录用

**核心判断**：本稿已达到投稿门槛，但有 X 处必修问题与 Y 处建议改进项。
```

## 五、修改优先级表

```markdown
## 修改优先级清单

| 优先级 | 修改项 | 预估工作量 | 影响 |
|---|---|---|---|
| 投稿前必须 | F1：摘要压缩至 280 字以内 | 30 分钟 | 符合期刊要求 |
| 投稿前必须 | F2：参考文献 X 条编号错位 | 1 小时 | 编辑部硬关 |
| 强烈建议 | M1：理论框架在第 3 章实质调用 | 半天 | 理论挂旗扣分项 |
| 强烈建议 | M2：英文摘要打磨 | 1 小时 | 国际读者印象 |
| 建议 | S1：第 2 章语体微调 | 30 分钟 | 整体打磨 |
| 建议 | S2：参考文献备注链接失效 | 15 分钟 | 抗复核 |
```

## 六、自评对话开场模板

当用户说"看看这一稿还有什么问题"，按以下流程响应：

```
1. 读稿（不输出，自己看完）
2. 加载相关 references（H-banned-words.md、本文件）
3. 跑扫描脚本（如本地能跑）
4. 切换四专家透镜，逐项扫描
5. 跑一票否决清单
6. 输出：
   - 总体判断（投稿建议：直接投 / 小修后投 / 大修后投 / 不建议投）
   - 致命问题清单（如有）
   - P1 / P2 清单
   - 终审评分表（如已成熟）
   - 修改优先级清单
   - 下一步建议（问用户要不要按此清单逐项处理）
```

## 七、自评频率建议

| 稿件状态 | 自评频率 |
|---|---|
| v1–v3 初稿 | 不需要全面自评，重点看结构 |
| v4–v8 结构修复 | 每 2-3 轮做一次结构性自评 |
| v9–v15 内容修复 | 每轮做内容性自评，每 3 轮做一次全面自评 |
| v16+ 打磨 | 每轮全面自评 |
| 投稿前 | 跑完整的一票否决清单 + 终审评分 |

---

## 八、不要假装审稿

如果对某个判断没有把握（如某个理论概念是否使用得当），**承认不确定**，不要假装专家。可以说：

> "笔者认为这里有概念错位的风险，但 Etzkowitz 三螺旋的边界条件这一表述的准确性需要进一步核实原始文献，建议查阅 Ranga & Etzkowitz (2013)。"

而不是直接断言"这里错了"。
