# 01. 选题立项

从用户的一句话（"我想写关于 X 的论文"）到一个可执行的研究问题，这一步往往决定整篇文章的命运。

---

## 一、首先问清楚三件事

任何选题对话都必须先问清这三件：

1. **目标期刊**：是哪本 C 刊？这决定字数、体例、写作语气、可接受的主题范围。
2. **核心问题**：用一句话能说清楚要回答什么吗？如果不能，先停在这里把问题问清楚。
3. **现有材料**：手头已经有什么？已发表论文、数据、田野材料、政策文件？这决定从哪里起手。

不要在没确认这三件事的情况下，直接讨论"理论框架用什么"或"怎么写引言"——那是后置的问题。

---

## 二、选题质量的四维评估

每个候选选题用以下四维评估：

| 维度 | 关键问题 | 判定方法 |
|---|---|---|
| 学术新意 | 已有研究覆盖了吗？空白在哪？ | 在 CNKI、Google Scholar 各搜 3-5 个关键词组合，看现有论文规模和切入角度 |
| 政策时效 | 当下有政策窗口吗？三年后仍重要吗？ | 看政府工作报告、五年规划、重大法规，是否在讨论这个议题 |
| 可操作性 | 能在 6-12 个月内做完吗？数据可得吗？ | 列出关键数据 / 案例 / 文献清单，看是否都能拿到 |
| 与作者匹配 | 作者的既有积累能用上吗？ | 看过去发表的论文、参与的项目，是否能形成连续性 |

四维都打 ★★★★ 以上可立项；任一维 ★★ 以下要重新评估。

---

## 三、问题意识的打磨

### 3.1 三个层次的问题

一个好的研究问题应该能在以下三个层次表达：

1. **大问题（field-level）**：本研究所属的学术领域关心什么大问题
2. **中问题（topic-level）**：本研究聚焦于大问题中的哪一个子议题
3. **小问题（specific research question）**：本研究的具体研究问题

示例：

```
大问题：发达经济体如何应对产业链重组对人才结构的冲击？
中问题：在集成电路领域，发达经济体如何重构人才培养体系？
小问题：哪些政策工具、组织结构和培养机制的组合，能解释五个主要案例的路径分化？
```

每篇论文要在引言中清晰区分这三个层次。

### 3.2 问题的可证伪性

学术问题必须是 **可证伪的**。判断方法：

- 如果回答是"是 / 否"或"有 / 无"——可证伪
- 如果回答是"重要 / 不重要"——不可证伪，重写
- 如果回答是"应该 / 不应该"——是政策建议，不是学术问题，要么重写为"为什么会出现 X""X 的条件是什么"，要么单独做政策研究

### 3.3 问题与方法的匹配

| 问题类型 | 适配方法 |
|---|---|
| "为什么会出现 X" | 案例研究、历史制度分析、过程追踪 |
| "X 和 Y 是什么关系" | 量化研究、回归分析 |
| "X 在不同条件下如何运作" | 比较研究、QCA |
| "X 是什么样的" | 描述性研究、类型学构建 |
| "怎么应对 X" | 政策研究、行动研究 |

问题和方法不匹配时，要么改问题，要么改方法，不能强行配对。

## 四、题目拟定

### 4.1 C 刊题目的常见结构

```
结构 A：主题 + 副标题（最常用）
    例：发达经济体如何战略性重构集成电路人才培养体系——基于"政策—组织—培养"框架的比较研究

结构 B：研究问题式
    例：法案约束如何塑造产业人才培养体系？

结构 C：现象 + 解释式
    例：集成电路人才培养的国际差异及其制度基础
```

### 4.2 题目质量的判断

| 检查 | 通过标准 |
|---|---|
| 是否表达了研究问题 | 读完题目知道在问什么 |
| 是否提示了方法 | 副标题或题目中暗示了方法（比较、案例、量化等） |
| 是否吸引目标读者 | 同领域学者看到会想读 |
| 字数控制 | 主标题 ≤ 20 字，副标题 ≤ 25 字 |
| 是否有名词堆叠 | 避免四层以上的定语堆叠（如"基于政策—组织—培养因果链的发达经济体集成电路人才培养体系战略性重构比较研究"）|

### 4.3 题目雷区

- "试论 X"——口语化，C 刊基本不用
- "X 研究"——信息量为零的尾词
- "对 X 的思考"——非学术化
- "X 的若干思考"——非学术化
- "X 探析 / 探讨 / 探究"——空动词
- 包含"创新""探索""新时代"等政策修辞的，谨慎使用

## 五、立项产出物

完成立项阶段时，应有以下五份文件：

### 5.1 选题立项书（500-800 字）

```markdown
# 选题立项书

## 一、基本信息
- 题目（暂定）：
- 目标期刊：
- 预计字数：
- 完成时间：

## 二、研究问题
- 大问题：
- 中问题：
- 小问题：

## 三、文献缺口
- 已有研究覆盖范围：
- 本研究填补的空白：

## 四、理论资源
- 主要理论框架：
- 次要理论资源：

## 五、研究方法
- 主要方法：
- 数据 / 案例来源：

## 六、预期贡献
- 学术贡献：
- 政策贡献（如适用）：

## 七、风险评估
- 数据可获得性：
- 时间可控性：
- 政治敏感性：
```

### 5.2 文献追踪清单

最少包含：
- 5 篇直接相关的中文 C 刊论文
- 5 篇相关的英文期刊论文
- 3 篇相关理论文献（专著或经典论文）
- 5 份相关政策文件或一手资料

### 5.3 案例 / 数据清单（如适用）

- 比较研究：明确所有比较对象，标注数据可得性
- 量化研究：数据源、变量定义、样本范围
- 案例研究：具体案例、获取材料的渠道

### 5.4 框架草图

一张图（或一段文字），勾勒出研究的核心分析框架。这不是最终框架，但能在立项时让作者看到自己的思路是否成立。

### 5.5 大纲（粗版）

引言 → 文献 → 方法 → 实证 → 讨论 → 结论。每节列 3-5 个要点。

---

## 六、不要立项的情形

以下情形应建议用户**先不立项，回去补功课**：

- 大问题想不清楚，只能说"这个话题很重要"
- 现有文献完全不熟悉
- 没有任何一手材料或可靠数据来源
- 题目和方法明显错配
- 政治敏感性高但作者没有相应的处理能力
- 写作动机是"评职称需要发一篇"——这种情况应该选作者最熟悉的方向，不是当下热点
