# 05. 初稿撰写

完成立项、理论框架、方法设计后，进入初稿撰写阶段。本文件给出从大纲到 v1 的完整流程。

---

## 一、初稿撰写的总策略

**先骨架，后血肉**。错误的做法：

- 从头到尾写一遍——后面会发现前面的论证支撑不住
- 先写引言（最难写）——前期的引言一定会大改
- 想到哪写到哪——结构会失控

正确的做法：

```
步骤 1：写一份 detailed outline（每节列要点）
步骤 2：先写实证 / 经验材料章节（最有把握的部分）
步骤 3：再写理论框架节
步骤 4：再写文献综述节
步骤 5：再写讨论 / 比较 / 分析节
步骤 6：再写方法节
步骤 7：最后写引言（前面都定了，引言才有依据）
步骤 8：最后写摘要（全文成型后才有可总结的内容）
步骤 9：最后写结语（与摘要对应）
步骤 10：最后写题目（前面都定了，题目才能精准）
```

不一定每篇都严格按此顺序，但**引言、摘要、题目放最后**这一原则不要破。

---

## 二、Detailed Outline 的格式

```markdown
# [论文题目（暂定）]

## 引言（约 1500-2000 字）
[写最后]

## 一、文献综述（约 2000 字）
- 主题 A 的研究进展：核心论点 + 关键文献 3-5
- 主题 B 的研究进展：核心论点 + 关键文献 3-5
- 文献定位声明：本研究的位置

## 二、理论框架（约 1500-2000 字）
- 主理论：X 理论
- 辅助理论：Y、Z
- 理论使用方案：哪一节用什么做什么

## 三、研究方法（约 1500 字）
- 方法选择：结构化聚焦比较
- 案例选择：5 个 + 选择理由
- 操作化：四维编码矩阵
- 分析步骤
- 局限

## 四、政策维度（约 2500 字）
- 案例 1：核心政策工具 + 1-2 段证据
- 案例 2：...
- 案例 3：...
- 案例 4：...
- 案例 5：...
- 维度小结

## 五、组织维度（约 2500 字）
[同上]

## 六、培养机制维度（约 2500 字）
[同上]

## 七、比较与解释（约 2000 字）
- 三组转向（共性）
- 三组解释变量（差异）
- 理论回应

## 八、政策启示（约 1500 字）
- 启示 1：借鉴方向 + 制度前提 + 适配性判断
- 启示 2：...
- 启示 3：...

## 结语 + 局限（约 500 字）
[写最后]

## 摘要（中文 ≤ 288 字，英文 250-350 词）
[写最后]
```

注意每节字数预算。预算超过 20% 就停下来反思是否在堆砌。

## 三、各节的写作技法

### 3.1 引言（最后写）

引言的功能：在 5 分钟内让审稿人决定要不要继续读。

结构（约 1500-2000 字）：

```
第 1 段（200-300 字）：现象 / 问题的提出
- 用具体数据或事件开篇，不要用空话
- 让读者立即看到"这个话题值得研究"

第 2 段（200-300 字）：国内背景
- 中国 C 刊投稿这一段不可省
- 引用中国相关政策（"十五五"规划等）
- 说明国内对此话题的需求

第 3 段（300-400 字）：已有研究简述与缺口
- 不展开（文献综述节会展开）
- 只说核心结论：哪些已有研究覆盖了什么、缺口在哪

第 4 段（200-300 字）：研究问题
- 明确写出本研究要回答的具体问题
- 区分大问题 / 中问题 / 小问题

第 5 段（200-300 字）：理论资源与方法概述
- 简要说明用什么理论、什么方法
- 不展开（后面有专门节）

第 6 段（150-250 字）：本研究的贡献与论文结构
- 1-2 句说贡献
- 1-2 句说论文结构（这是元叙述，允许）
```

引言的雷区：

- 第 1 段从"在……背景下"开始——AI 痕迹
- 大量套话堆叠——"具有重要的理论与实践意义"
- 文献综述塞进引言——展开到 1000 字以上
- 没有具体研究问题——读完不知道要回答什么
- 没有数据 / 事件锚点——通篇抽象

### 3.2 案例分析节

每个案例 500-800 字，结构：

```
[案例 1：美国，约 600-800 字]

第 1 段（150-200 字）：背景与基本事实
- 简述该案例的政策背景、关键时间节点

第 2 段（200-300 字）：核心机制描述
- 用本研究的分析框架描述该案例
- 用理论的概念词

第 3 段（200-300 字）：分析与判断
- 解释为什么这样
- 与其他案例对照（如已写到后面案例）

第 4 段（可选，100-150 字）：限定或反例
- 该案例不适用本框架的部分
- 该案例的特殊性
```

案例并列节的雷区：

- 五个案例完全同构——AI 痕迹
- 每段开头都用"X 通过 Y 实现 Z"——重写
- 信息密度不一致（某些案例 200 字，某些 800 字）
- 没有具体数据 / 法规名 / 机构名——抽象
- 段尾必带"这意味着……"——AI 痕迹

### 3.3 比较与解释节

比较节不应只是"汇总前面"。它应该是分析的高峰。

结构：

```
第 1 段：共性的提炼
- 五案例（或主案例）共同呈现什么趋势
- 用 2-3 组"转向"或"共性"概括

第 2-4 段：差异的解释
- 为什么会出现路径分化
- 用结构变量解释（如：产业结构、政策能力、高校治理）
- 每个变量配 1-2 段分析

第 5 段：理论回应
- 本研究的发现对理论资源有何回应
- 是否对理论提出限定、修正或推进
```

### 3.4 政策启示节

中国 C 刊几乎都期待有政策启示节，但要写得有学理性。

每条启示按"借鉴方向—制度前提—适配性判断"三段式写：

```
**启示 1：[借鉴方向]**

[借鉴方向，1-2 句]

[制度前提，2-3 句：该做法在原案例中依赖的制度条件]

[适配性判断，2-3 句：我国是否具备相应条件，若不具备应做什么调整]
```

避免：

- 单向倡议（"应该……""必须……"）——空话
- 没有学理依据的建议
- 与正文论证脱节的启示

### 3.5 结语（最后写）

结语 300-500 字，结构：

```
第 1 段（150-200 字）：核心发现概括
- 1-2 句说研究问题
- 2-3 句说核心发现

第 2 段（100-150 字）：贡献声明
- 学术贡献
- 政策含义

第 3 段（100-150 字）：研究局限
- 至少 2 条具体局限
- 后续研究方向
```

如果目标期刊（如《中国高教研究》）不设独立结语节，把结语融入最后一节末尾。

### 3.6 摘要（最后写）

中文摘要按 C 刊体例 ≤ 288 字（具体看目标期刊）。

结构（按重要性排序）：

```
1. 研究问题（30-50 字）
2. 方法概述（30-50 字）
3. 核心发现（80-120 字）
4. 政策含义或贡献（30-50 字）
```

雷区：

- 摘要展开背景（C 刊摘要无背景段）
- 摘要展开方法（方法节会写）
- 摘要展开文献综述
- 摘要用"本文""本研究"开头（虽然不严格禁止，但开头直接给结论更有力）

### 3.7 关键词

3-8 个，按"宽—窄"排序：

```
集成电路；半导体；战略性重构；工程教育；人才培养
```

第 1 个最宽（领域），最后 1 个最窄（本研究的特定概念）。

---

## 四、英文摘要

中文 C 刊几乎都要英文摘要。结构与中文摘要对应，250-350 词。

写作要点：

- 不直译中文摘要——英文学术表达有自己的节奏
- 句子要短，避免长定语
- 主动语态优先（"We compare……" 不是 "A comparison is conducted……"）
- 关键概念附中文（如 "strategic restructuring (战略性重构)"）

---

## 五、初稿成型的判断

什么时候算 v1 完成？

- 所有节均已写完（无 [待补] 标记）
- 引言、摘要、题目均已写
- 参考文献已初步整理（不必完美）
- 字数符合期刊要求（±20%）
- 全文通读一遍能读通

不要追求 v1 完美。v1 完美也只是 v1，后续仍有 v2 到 v30 的迭代。

---

## 六、初稿撰写时的风格底盘

参考 `A-style-guide.md`。重点：

- 第三层规则（结构）写时就要注意，否则后期改起来代价大
- 第二层规则（句式）写时尽量避免，但允许少量违例
- 第一层规则（词汇）写时可以宽松，最后扫一遍即可

不要边写边纠结词汇。词汇是最后清理的。

---

## 七、初稿撰写的速度

成熟作者写 C 刊论文的合理节奏：

| 阶段 | 时间 |
|---|---|
| Detailed outline | 1-2 天 |
| 实证 / 案例章节 | 5-7 天 |
| 理论 + 文献 + 方法 | 3-5 天 |
| 比较 + 启示 | 2-3 天 |
| 引言 + 结语 + 摘要 + 题目 | 1-2 天 |
| **合计（v1 完成）** | **12-19 天** |

之后还有 4-12 周的迭代。所以从立项到投稿，3-4 个月是常见周期。

---

## 八、写作中常见的卡点与对策

| 卡点 | 对策 |
|---|---|
| 某节写不下去 | 跳过这节，写其他能写的，回头再补 |
| 不确定理论概念用得对不对 | 标注 [理论待核]，继续写 |
| 不确定某个数据 | 标注 [数据待核]，先用占位符 |
| 论证觉得弱 | 标注 [论证待加强]，继续写 |
| 写到一半发现框架有问题 | 停下来，回到 detailed outline 调整框架 |

不要为了一个细节卡住整体进度。**写一稿的目标是把骨架立起来**，细节是后续迭代的事。
