# G. CNKI AIGC 检测应对

CNKI（中国知网）的 AIGC 检测是 C 刊投稿的实际门槛之一。本文件给出系统性的应对方案。

---

## 一、检测机制简述

CNKI 的 AI 检测主要依靠：

1. **语言模型困惑度（perplexity）**：句子是否"过于流畅"
2. **句子平滑度分析**：句间过渡是否"无缝衔接"
3. **结构模式识别**：段落是否"高度对称"
4. **词汇特征**：是否包含 AI 偏好词

**关键认知**：检测器主要捕获 **结构性** 和 **流利度** 特征，而不只是词汇。靠禁用词表降不到根本。

---

## 二、风险区间

业界经验值（CNKI 未公开阈值）：

| AI 率 | 风险等级 | 处理建议 |
|---|---|---|
| < 15% | 低 | 通常可直接投稿 |
| 15-30% | 中低 | 可投稿，编辑可能要求微调 |
| 30-40% | 中 | 投稿前需重点修改高风险段 |
| 40-60% | 高 | 大概率需重写若干段 |
| > 60% | 极高 | 大概率被退稿 |

不同期刊容忍度不同。顶级 C 刊（如《管理世界》）容忍度较低，建议控制在 20% 以下。

---

## 三、高风险区域识别

### 3.1 全文风险分布

```
| 区域 | 典型 AI 率 | 风险等级 |
|---|---|---|
| 引言 | 25-35% | 中 |
| 文献综述 | 20-30% | 中低 |
| 理论框架 | 25-35% | 中 |
| 方法 | 15-25% | 中低 |
| 案例分析 | 15-25% | 中低 |
| 比较与解释 | 30-40% | 中高 |
| 政策启示 | 35-45% | 高 |
| 结语 | 35-45% | 高 |
```

**结语和政策启示节**是 AI 检测最敏感的区域。

### 3.2 高风险段落的特征

- 论证完整、过渡自然、几乎无冗余 ← **反而是 AI 信号**
- 多段并列、句式同构
- 段尾必有"小结升华"
- "本研究 / 本文……"作为段首
- 单向论证，无转折或限定
- 抽象概念堆叠

---

## 四、降 AI 率的三层策略

### 4.1 第三层（最深，最有效）：结构与论证

#### 策略 1：打破段落对称性

```
反例：
> 美国通过 X 实现 Y。
> 日本通过 A 实现 B。
> 韩国通过 P 实现 Q。
> 台湾通过 M 实现 N。

正例：
> 美国选择 X 路径：[具体细节]。日本的做法不同——其将 A 嵌入 B 法案，
> 形成嵌套结构。韩国的特征是行政动员的高强度：[案例]。台湾则采取
> 专项立法的方式。
```

#### 策略 2：引入第一人称判断

```
反例：
> 综合上述分析，治理重心正由院校内部转向国家主导。

正例：
> 笔者认为，治理重心的这一移动并不彻底——院校层面的课程改革仍由
> 各高校自主推进。
```

#### 策略 3：加入具体细节

```
反例：
> 美国通过多项立法措施推动半导体人才培养。

正例：
> 美国 2022 年通过的 CHIPS and Science Act 划拨 528 亿美元用于半导体
> 制造与研发，其中明确要求接受 NOFO 资助的企业承诺培养指定数量的
> 半导体工程师。
```

#### 策略 4：保留摩擦感

```
反例：
> 三螺旋理论为本研究提供了有力的分析工具。

正例：
> 三螺旋理论为本研究提供了主要分析工具，但该理论原本基于发达经济体
> 高校创业活动的描述，应用于产业政策驱动的人才培养体系时，需对其
> "三方均衡"假设作适当限定。
```

#### 策略 5：删除元叙述

```
反例（元叙述句过密）：
> 本文将首先讨论政策维度，其次讨论组织维度，最后讨论培养机制维度。
> 在政策维度的分析中，本研究将……

正例：
> 政策维度的分析从五案例的政策工具类型识别开始。[直接进入实质]
```

### 4.2 第二层：句式与标点

详见 `A-style-guide.md` 第四部分。重点：

- 破折号不充当插入语
- "不仅……更……"句式控制
- 句首禁语全部删除
- 形式主语和被动语态减少

### 4.3 第一层（最表层，但仍要做）：词汇

详见 `H-banned-words.md`。重点：

- 硬禁用词归零
- 密度管理词控密度
- 立场词改为"笔者认为"

---

## 五、按区域的修改优先级

### 5.1 结语（最高优先级）

结语 300-500 字，是 AI 检测最敏感的区域，重点处理：

- 删除规范性收口（"综上所述""综合以上"）
- 加入第一人称判断（"笔者认为"）
- 至少 2 处具体研究局限
- 段落内允许转折
- 不强行"升华意义"

### 5.2 政策启示节（次高优先级）

政策启示节通常 1500 字左右，每条启示按"借鉴方向—制度前提—适配性判断"三段式写：

- 不做单向倡议
- 每条启示有制度条件分析
- 适配性判断有具体依据

### 5.3 比较与解释节（中等优先级）

比较节的"三组转向"总结段是高风险点：

- 打破排比结构
- 插入带具体数据 / 案例的细节句
- 加入限定或反例

### 5.4 引言（中等优先级）

引言改写重点：

- 第一段不用"在……背景下"开头
- 删除套话堆叠
- 研究问题具体明确

## 六、降 AI 率的工作流

### 6.1 自测阶段

```
1. 跑禁用词扫描（H-banned-words.md 的脚本）
2. 跑结构对称性扫描（人工通读）
3. 估算各区域 AI 风险等级
4. 列出修改优先级
```

### 6.2 修改阶段

```
按优先级处理（结语 → 启示 → 比较 → 引言 → 其他）：
1. 结构层重写（如需要）
2. 句式层调整
3. 词汇层清理
```

### 6.3 验证阶段

```
1. 重新跑扫描脚本
2. 通读高风险段，确认有摩擦感
3. 提交 CNKI AIGC 检测（如可获取）
4. 根据检测结果再迭代
```

## 七、不要做的事

### 7.1 不要为了降 AI 率而牺牲学术品质

某些"降 AI 率"建议会伤害论文：

- 故意写错别字：禁
- 故意用怪异表达：禁
- 删除关键论证：禁
- 加入无关内容：禁

降 AI 率必须以"提升论文质量"为前提，不可本末倒置。

### 7.2 不要靠改字符级特征

- 全角半角混用：无效
- 加零宽字符：无效，且涉及学术不端
- 同义词机械替换：无效

CNKI 检测对这些 trick 已经免疫。

### 7.3 不要重复使用相同改法

如果对全文都用"笔者认为"，反而成为新的模式特征。各种降 AI 化技巧要轮换使用。

---

## 八、降 AI 率与提升论文质量的关系

经验观察：**真正高质量的学术写作天然 AI 率低**。原因：

- 真正深入的分析有摩擦感
- 真正原创的论证有边界限定
- 真正扎实的案例有具体细节
- 真正成熟的作者有判断节奏

所以最佳策略是：**写一篇真正好的论文，AI 率会自动低**。降 AI 率不是单独的工作，是论文质量提升的副产品。

---

## 九、检测后的应对

如已知 AI 率超标，按以下流程处理：

### 9.1 定位高风险段

CNKI 检测报告会高亮高风险段落。先聚焦高亮区域。

### 9.2 评估每个高亮段

| 高亮原因 | 处理 |
|---|---|
| 段落对称 | 重写打破对称 |
| 句式同构 | 改写至少一半句式 |
| 词汇 AI 化 | 替换为本节列出的替代词 |
| 论证无摩擦 | 加入转折或限定 |
| 段尾升华 | 删除或改为具体判断 |

### 9.3 不要全文重写

如果某段被高亮，只重写该段。重写全文容易引入新问题。

### 9.4 重写后再检测

迭代直至 AI 率落入安全区间。

---

## 十、特殊情况：明明是人写的也被检测

如果用户的内容**确实是自己写的**，但 AI 率检测仍偏高，原因可能是：

1. **写作风格本身偏 AI 化**（多年使用 AI 辅助形成的习惯）
2. **学术语言天然格式化**（部分误检不可避免）
3. **本文与 AI 训练语料风格重合**（如政策文件话语）

应对：

- 接受部分误检不可避免
- 尽量降低可控部分的 AI 特征
- 投稿时如有要求，可提交写作过程证据（草稿、批注、引用文献截图等）

## 十一、本节扫描脚本

```bash
#!/bin/bash
# AI 风险扫描

FILE=$1

echo "=== 第一层：词汇（参考 H-banned-words.md）==="
bash scan-banned.sh "$FILE"

echo ""
echo "=== 第二层：句式 ==="
echo "破折号位置（人工判断是否插入语）："
grep -nE '——' "$FILE" | head -20
echo ""
echo "句首禁语："
grep -nE '^[。\s]*(值得注意|不难发现|显而易见|综上|综合以上|总而言之|由此可见)' "$FILE"
echo ""

echo "=== 第三层：结构 ==="
echo "元叙述密度（全文 ≤ 5）："
grep -cE '本文|本节|下文将|上文已' "$FILE"
echo ""
echo "段尾'升华句'候选（人工判断）："
grep -nE '(这意味着|这一变化|这一转变).{0,30}(转向|表明|意味)' "$FILE" | head -10
echo ""
echo "并列段同构候选（人工判断）："
# 提取每段开头几字，看是否重复
awk 'length > 50 {print substr($0, 1, 30)}' "$FILE" | sort | uniq -c | sort -rn | head -10

echo ""
echo "=== 立场词 ==="
grep -nE '研究表明|本文认为|我们认为' "$FILE"
```

---

## 十二、底线

如果一篇论文从核心理论、案例分析到论证全部由 AI 生成，**不应投稿**。这不是降 AI 率技术问题，而是学术伦理问题。

本节的所有方法只适用于：

- 作者主导论证、AI 辅助写作的稿件
- 作者自己写但风格偏 AI 化的稿件

不适用于完全 AI 生成的稿件。
