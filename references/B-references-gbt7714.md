# B. 参考文献体系管理（GB/T 7714-2015）

参考文献管理在多轮迭代中最容易失控。本文件给出从条目格式到全文体系一致性的完整方案。

---

## 一、GB/T 7714-2015 格式速查

### 1.1 各类文献的标准格式

```
期刊论文 [J]
[序号] 作者. 题名[J]. 刊名, 年, 卷(期): 起止页码.

专著 [M]
[序号] 作者. 书名[M]. 版次（第1版不写）. 出版地: 出版者, 年: 页码.

学位论文 [D]
[序号] 作者. 题名[D]. 保存地: 保存单位, 年.

会议论文 [C]
[序号] 作者. 题名[C]//论文集名. 出版地: 出版者, 年: 页码.

报纸文章 [N]
[序号] 作者. 题名[N]. 报纸名, 出版日期(版次).

电子文献 [EB/OL] / [J/OL]
[序号] 作者. 题名[EB/OL]. (发布日期)[引用日期]. URL.

政府报告 [R/OL]
[序号] 机构. 报告名[R/OL]. (发布日期)[引用日期]. URL.

政策文件 [Z] / [Z/OL]
[序号] 发布机构. 文件名[Z/OL]. (发布日期)[引用日期]. URL.

标准 [S]
[序号] 标准编号 标准名称[S]. 出版地: 出版者, 年.
```

### 1.2 文献类型标识对照

| 标识 | 类型 | 用法 |
|---|---|---|
| [M] | 专著 | 整本书 |
| [J] | 期刊论文 | 学术期刊论文 |
| [D] | 学位论文 | 硕士、博士论文 |
| [C] | 会议论文 | 会议论文集中的单篇 |
| [N] | 报纸 | 报纸文章 |
| [R] | 报告 | 研究报告、机构报告 |
| [S] | 标准 | 国标、行标 |
| [P] | 专利 | 专利文献 |
| [EB/OL] | 电子公告 / 在线 | 网络资源 |
| [J/OL] | 在线期刊 | 网络期刊 |
| [R/OL] | 在线报告 | 网络报告 |
| [Z] | 其他 | 其他类型 |

## 二、作者格式

### 2.1 中文作者

```
单作者：李明
双作者：李明, 王芳
三作者：李明, 王芳, 张华
四人及以上：李明, 王芳, 张华, 等
```

注意：

- 中文姓名作者之间用半角逗号 `,` 加半角空格
- 不空格姓和名（"李明"不是"李 明"）
- 4 人及以上写前 3 人加 "等"

### 2.2 英文作者

```
单作者：SMITH J
双作者：SMITH J, JOHNSON A
三作者：SMITH J, JOHNSON A, BROWN R
四人及以上：SMITH J, JOHNSON A, BROWN R, et al
```

注意：

- 姓在前，全大写
- 名取首字母，无点
- 多名首字母连写（如 "BROWN R J"）
- 4 人及以上写前 3 人加 "et al"

### 2.3 机构作者

```
中文：中华人民共和国教育部
英文：U.S. Department of Commerce
```

机构作者完整列出，不缩写。

## 三、典型条目示例

### 3.1 中文期刊论文

```
[1] 管开轩, 余江, 周建中, 等. 高水平科技自立自强下我国集成电路人才培养
"痛点"与对策[J]. 中国科学院院刊, 2023, 38(2): 336-348.
```

要点：

- 作者间逗号 + 空格
- 题名后立即跟 [J]
- 刊名后逗号
- 年、卷(期): 页码
- 页码间用半角短横线 `-`

### 3.2 英文期刊论文

```
[2] ETZKOWITZ H, LEYDESDORFF L. The dynamics of innovation: from National
Systems and "Mode 2" to a Triple Helix of university-industry-government
relations[J]. Research Policy, 2000, 29(2): 109-123.
```

要点：

- 作者格式：姓全大写，名首字母
- 题名首字母大写，专有名词大写（其余小写）
- 刊名按惯例（如 Research Policy 大写）
- 卷(期): 页码

### 3.3 中文专著

```
[3] 林健. 卓越工程师教育培养计划研究[M]. 北京: 清华大学出版社, 2013: 45-67.
```

### 3.4 英文专著

```
[4] HALL P A, SOSKICE D. Varieties of Capitalism: The Institutional
Foundations of Comparative Advantage[M]. Oxford: Oxford University Press,
2001: 1-68.
```

### 3.5 政策文件

```
[5] 中华人民共和国国务院. 中华人民共和国国民经济和社会发展第十五个
五年规划纲要[Z]. 2026.
```

涉台政策文件：

```
[6] 台湾地区相关主管部门. "重点领域产学合作及人才培育创新条例"[Z/OL].
(2021)[2026-03-01]. https://...
```

注意涉台规则：

- 颁布主体改为"台湾地区相关主管部门"或"台湾地区行政管理机构"
- 法规名加引号去"国家"二字
- 不出现"台湾"作为独立国家

### 3.6 政府报告 / 在线报告

```
[7] U.S. Department of Commerce. CHIPS Act Notice of Funding Opportunity:
Commercial Fabrication Facilities[R/OL]. (2023-02-28)[2026-03-05].
https://www.nist.gov/chips/...
```

### 3.7 网络资源

```

[2026-03-12]. https://moe.go.kr/...
```

## 四、引用与参考文献的对齐

### 4.1 顺序编码制

GB/T 7714-2015 采用顺序编码制：参考文献按 **在正文首次出现的顺序** 编号。

注意：

- 不按作者姓氏字母顺序
- 不按发表时间顺序
- 按首次出现顺序

### 4.2 多轮修订后的常见问题

| 问题 | 描述 |
|---|---|
| 缺号 | 文末参考文献编号不连续 |
| 引而未列 | 正文有 [N] 但文末参考文献中无 [N] |
| 列而未引 | 文末有 [N] 但正文中未引用 |
| 顺序错乱 | 文末编号不按正文首次出现顺序 |

每轮修订都需要检查这四点。

### 4.3 自动化检查脚本

```python
#!/usr/bin/env python3
"""参考文献体系完整性检查脚本"""

import re
import sys

def check_references(filepath):
    with open(filepath, 'r', encoding='utf-8') as f:
        text = f.read()
    
    # 分离正文和参考文献区
    # 假设参考文献区从"参考文献"或第一条 [1] 后开始
    parts = re.split(r'参考文献\s*\n', text, maxsplit=1)
    if len(parts) < 2:
        body = text
        refs = ""
    else:
        body, refs = parts[0], parts[1]
    
    # 提取正文中的引用编号
    body_citations = sorted(set(int(x) for x in re.findall(r'\[(\d+)\]', body)))
    
    # 提取参考文献区的编号（行首形如 [N] 的）
    ref_nums = sorted(set(int(x) for x in re.findall(r'^\[(\d+)\]', refs, re.M)))
    
    # 检查
    body_set = set(body_citations)
    ref_set = set(ref_nums)
    
    cited_but_not_listed = body_set - ref_set
    listed_but_not_cited = ref_set - body_set
    
    print(f"正文引用编号: {body_citations}")
    print(f"参考文献编号: {ref_nums}")
    print(f"引而未列: {sorted(cited_but_not_listed)}")
    print(f"列而未引: {sorted(listed_but_not_cited)}")
    
    # 检查参考文献编号是否连续
    if ref_nums:
        expected = list(range(1, max(ref_nums) + 1))
        missing = sorted(set(expected) - ref_set)
        print(f"缺号: {missing}")
    
    # 统计文献类型
    journals = len(re.findall(r'\[J\]', refs))
    monographs = len(re.findall(r'\[M\]', refs))
    print(f"\n期刊论文 [J]: {journals}")
    print(f"专著 [M]: {monographs}")
    print(f"总文献数: {len(ref_nums)}")

if __name__ == "__main__":
    check_references(sys.argv[1])
```

保存为 `check_refs.py`，每轮修订后跑一次。

### 4.4 重编号脚本（按首次出现顺序）

```python
            first_appear[n] = match.start()

```

注意：

- 重编号时必须先用占位符（如 `<<TEMP_42>>`）再替换为新号，避免重复替换
- 同时处理正文引用和参考文献条目本身
- 处理多个引用合并的情况：`[1, 3, 5]` 或 `[1-3]`

## 五、文献核实

### 5.1 中文文献核实

通过 CNKI 或 Wanfang 检索：

```
关键字段：
- 题名（容易抄错）
- 作者列表（多作者容易漏）
- 期刊全称（容易写简称）
- 卷期号（最易出错）
- 页码
```

不确定时标注：

```
[1] 管开轩, 余江, 周建中, 等. 高水平科技自立自强下我国集成电路
人才培养"痛点"与对策[J]. 中国科学院院刊, 2023, 38(2): 336-348.
（注：卷期号可能不准确，建议在 CNKI 核实）
```

### 5.2 英文文献核实

- Google Scholar 查 DOI
- Web of Science 查准确卷期页
- 出版社官网（如 Elsevier、Wiley）查最权威

DOI 标注（GB/T 7714-2015 允许）：

```
[2] ETZKOWITZ H, LEYDESDORFF L. The dynamics of innovation[J].
Research Policy, 2000, 29(2): 109-123. DOI:10.1016/S0048-7333(99)00055-4.
```

### 5.3 政策文件核实

- 政府门户网站为权威源
- 链接易失效，注明检索路径作为备份：

```
[3] U.S. Department of Commerce. CHIPS Act NOFO[R/OL]. (2023-02-28)
[2026-03-05]. https://www.nist.gov/chips/...
（注：如链接失效，可在 NIST 官网检索关键词 "CHIPS Act NOFO"）
```

### 5.4 BAI（韩国审计院）等特殊源

- BAI 禁止自动抓取
- 详情 URL 用 srno 参数：`https://www.bai.go.kr/...?srno=XXXX`
- 引用时手动核实 srno 或留存检索关键词：

```
[4] 韩国审计院. 半导体人才培养审计报告[R/OL]. (2023)[2026-03-12].
https://www.bai.go.kr/bai/result/branch/list/.
（注：因 BAI 官网限制自动检索，具体报告页可在 BAI 官网检索关键词
"반도체 인재양성" 获取）
```

## 六、参考文献的中英比例

C 刊投稿建议：

| 类型 | 比例 |
|---|---|
| 中文期刊论文 [J] | 30-50% |
| 英文期刊论文 [J] | 20-35% |
| 中英学术专著 [M] | 10-20% |
| 政策文件 / 报告 [R/OL] [Z/OL] | 10-25% |
| 网络资源 [EB/OL] [N/OL] | ≤ 10% |

中文期刊文献偏少（< 30%）容易被认为"脱离本土学术语境"。

## 七、参考文献节的格式

文末参考文献节的格式：

```markdown
## 参考文献

[1] 第一条参考文献完整内容...
[2] 第二条参考文献完整内容...
[3] 第三条参考文献完整内容...
...
```

注意：

- 节标题用"参考文献"（不用"References"）
- 每条独占一段（或多行换行）
- 不同条目之间空一行（视期刊体例）
- 编号与文献之间无空格："[1] 作者..." 不是 "[1]  作者..."

## 八、特殊情况

### 8.1 多个引用合并

正文中：

```
[1, 3, 5]   或   [1-3]   或   [1][3][5]
```

具体用哪种看期刊体例。多数用 `[1, 3, 5]` 或 `[1][3][5]`。

### 8.2 同一文献多次引用

同一文献多次引用，编号保持一致：

```
正文：...如 X 研究所示[5]...，而 Y 研究[5]进一步表明...
```

不要为同一文献分配两个编号。

### 8.3 引用页码

某些情况下需引用具体页码：

```
正文：[5]156-160
```

GB/T 7714-2015 允许，但多数期刊用"夹注"或"脚注"标注页码，不在编号后直接加。

### 8.4 转引

不可避免的转引：

```
[6] X 著, Y 译. 题名[M]. 出版地: 出版者, 年: 页码. 转引自 Z. 文章名[J].
刊名, 年, 卷(期): 页码.
```

控制在 2-3 处以内。

## 九、参考文献错误的分类与修复

| 错误 | 修复方法 |
|---|---|
| 缺号 | 重新连续编号，需同时改正文引用 |
| 引而未列 | 补全条目或删除引用 |
| 列而未引 | 删除条目或在正文中加引用 |
| 顺序错乱 | 用重编号脚本按首次出现顺序重排 |
| 格式不一致 | 统一为 GB/T 7714-2015 |
| 信息不完整 | 补全（必要时标"可能不准确"） |
| 重复条目 | 合并为一条，统一编号 |

## 十、修订过程中的参考文献管理

### 10.1 增加新文献时

- 不要直接插到中间编号——会打乱后续
- 暂时编为最大号 +1，待修订完成统一重排
- 或用占位符（如 `[NEW-1]`）插入正文，最后批量替换

### 10.2 删除文献时

- 同时检查正文中所有相关引用是否需删除
- 不要留下"引而未列"的孤儿引用

### 10.3 每轮修订后

跑 `check_refs.py`，确保：

- 无缺号
- 无引而未列
- 无列而未引
- 编号按首次出现顺序

## 十一、参考文献的最终审查

投稿前必做：

```bash
# 跑检查脚本
python check_refs.py manuscript.docx

# 手工抽查
# 1. 随机抽 5 条文献，到 CNKI / Google Scholar 验证
# 2. 抽查所有政策文件 URL 是否可访问
# 3. 检查所有作者姓名拼写
# 4. 检查所有期刊名拼写
# 5. 检查所有 DOI（如有）
```

参考文献质量是审稿人对论文严谨性的第一印象。值得花 2-3 小时认真做。
