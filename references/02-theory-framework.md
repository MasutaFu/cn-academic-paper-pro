# 02. 理论框架搭建

C 刊论文中"理论挂旗、正文失联"是高频扣分项。本文件给出搭建理论框架时如何让理论真正参与正文分析的方法。

---

## 一、理论框架的两种典型病症

### 1.1 挂旗病

引言罗列三五个理论（"本文综合运用 X 理论、Y 框架、Z 视角"），正文几乎不再提及。审稿人会判定为"理论装饰"。

### 1.2 单理论垄断病

引入一个理论后全盘照搬其概念体系，把经验材料硬塞进理论框架里，看不到经验对理论的回应或推进。

---

## 二、判断理论是否"实质参与"

一个理论在论文中实质参与的标志：

| 标志 | 判断方法 |
|---|---|
| 用了理论的概念词 | 该理论的核心术语在正文出现并被用于描述案例 |
| 用了理论的分析逻辑 | 分析步骤遵循该理论的因果或机制模型 |
| 看到了理论的边界 | 经验材料展示了理论的不适用区域 |
| 推进或修正了理论 | 结论部分对理论提出了限定、补充或修正 |

四条中至少满足两条，才算"实质参与"。只满足第一条不够，那是"借词"。

---

## 三、理论框架的搭建顺序

不是"先选理论，再找材料"，也不是"只看材料，不用理论"。正确顺序：

```
步骤 1：明确研究问题
步骤 2：判断问题类型（描述性 / 解释性 / 比较性 / 规范性）
步骤 3：列出能回答这类问题的候选理论（3-5 个）
步骤 4：评估每个候选理论的适配度
步骤 5：选定一个主理论 + 1-2 个辅助理论
步骤 6：拟定理论与正文的对应方案（哪一章用什么理论做什么工作）
步骤 7：写理论框架节
```

步骤 6 是关键。理论框架节写完就要能预判正文每一节都会用哪些理论概念。

## 五、理论框架节的写作模板

理论框架节（通常在引言之后、方法节之前，1500-2500 字）的结构：

```markdown
## X. 理论框架

### X.1 已有理论资源

[200-400 字：简述与本研究相关的理论资源，不展开]

### X.2 主理论的选择与适用

[400-600 字：介绍主理论，说明为什么这是本研究的合适理论]
- 主理论的核心概念
- 适用边界
- 已有研究中的应用

### X.3 辅助理论的引入

[400-600 字：介绍辅助理论，说明其与主理论的关系]
- 辅助理论补足主理论的哪些空白
- 辅助理论的核心概念

### X.4 本研究的理论使用方案

[200-400 字：说明各理论在正文哪部分起什么作用]
- 在引言论证部分：用 X 理论做 Y 工作
- 在第一章 / 政策维度：用 Y 理论做 Z 工作
- 在第二章 / 组织维度：用 X 理论做 W 工作
- ...
- 在结论部分：对理论的回应或推进
```

### 5.1 X.4 的重要性

X.4 不是可选项。它是审稿人判断"理论是否实质参与"的关键。明确写出"哪节用哪个理论做什么"，等于给自己一个执行清单。

### 5.2 理论使用方案示例

```
本研究的理论使用方案如下。第一部分（政策维度）以政策工具组合理论
（Rogge & Reichardt 2016）为主要分析工具，识别五案例的政策工具类型组合
及其内在一致性。第二部分（组织维度）以三螺旋理论（Etzkowitz）和
混合组织（hybrid organization）概念为分析工具，刻画各案例中政府—产业—大学
的关系结构。第三部分（培养机制维度）以技能形成制度（Thelen 2004; Busemeyer
& Trampusch 2012）为分析工具，识别培养机制背后的制度逻辑。第四部分（比较）
以资本主义多样性（Hall & Soskice 2001）和国家创新体系（Freeman; Lundvall）
为解释变量，分析案例间路径分化的制度基础。
```

写到这种程度，正文就有了"理论分布"的明确蓝图。

## 六、如何避免"挂旗"

### 6.1 在每章开头明确理论调用

每章 1500-3000 字，开头第一段应该说明本章使用哪些理论概念。例：

> 本节以政策工具组合理论为分析框架，关注五案例在政策工具类型选择上的差异及其内在一致性。Rogge 和 Reichardt（2016）将政策工具组合的一致性区分为目标一致性、工具间一致性和稳定性三个层面，本节将以此为标准对五案例进行编码。

### 6.2 在分析中使用理论的概念词

```
反例（不用理论词）：
> 美国通过多种政策手段推动半导体产业发展。

正例（用理论词）：
> 美国采取了"法案约束型"政策工具组合：以 CHIPS Act 提供的财政补贴
> 为核心，配合 NSTC 协调机制和劳动部技能联盟的执行配套，三种工具
> 在目标一致性上达成了较高水平，但工具间一致性仍受联邦—州层级
> 分工的制约。
```

### 6.3 在章节小结部分对理论做出回应

每章小结 100-200 字，要在理论层面给出一个判断：

```
本节的发现对政策工具组合理论提供了两点经验材料的回应：
第一，工具间一致性并非自动达成；
第二，"法案约束型"组合在产业转型期具有特殊的协调优势，
但这一优势是否能跨越政治周期，需要后续观察。
```

## 七、理论之间的关系

如果用了多个理论，必须说清它们之间的关系：

| 关系 | 表达方式 |
|---|---|
| 并列 | "本研究综合运用 X 与 Y，分别用于分析……" |
| 主辅 | "以 X 为主理论框架，以 Y 补足 X 在……方面的局限" |
| 对话 | "本研究借助 X 与 Y 的对话，揭示……" |
| 整合 | "本研究尝试整合 X 与 Y，构建……的分析框架" |

不能让多个理论"漂浮"在文中，互不关联。

---

## 八、本土理论的处理

国内学者发展的理论资源不应被忽视。在 C 刊投稿中适当引用本土理论或本土学者对国际理论的应用，能显著提升"学术对话感"。

- 教育研究：潘懋元、刘海峰、邬大光、史秋衡、阎光才等
- 科技政策：贾根良、薛澜、孟庆国等
- 公共管理：朱旭峰、马亮、宋华琳等
- 高等工程教育：林健、夏建国、王孙禺等

具体引用哪些，按研究主题在 CNKI 搜索本主题的代表文献。

## 九、理论选择的常见错误

1. **理论越多越好**——错。三个以上理论很难都实质参与。
2. **必须用前沿理论**——错。经典理论用得好比追新潮安全。
3. **理论决定一切**——错。理论是工具，问题决定理论选择。
4. **理论框架节越长越好**——错。1500-2500 字够用，超过 3000 字一般是过度。
5. **不用理论也能写**——错。C 刊几乎不接受无理论框架的论文。

---

## 十、当用户的论文已经写了一稿，但理论挂旗时

按以下流程帮助修正：

1. 列出现有论文中提到的所有理论
2. 在正文中 grep 这些理论的关键词，统计分布
3. 如果某个理论只在引言出现，问用户：要么删除（理论不必要），要么补强（在正文 X 章实质调用）
4. 如果保留，按 X.4 的方法做一份"理论使用方案"
5. 按方案在正文每章嵌入理论概念词
6. 在每章小结对该理论做出回应

不要"事后挂旗"——即在引言加一段理论介绍但正文不动。审稿人能看出来。
