# A. 去 AI 化与学术语体详则

本文件是 SKILL.md 第四部分"风格底盘"的展开。所有规则按"原理 → 规则 → 反例 → 正例"四段式组织，方便对照修改。

---

## 一、底层心智模型

学术中文应该读起来像**一位经验丰富的中文学者在认真说话**，而不是：
- 翻译件（句法上是英文的中文）
- AI 生成（流畅但空洞、对称、抽象）
- 政策文件话语（套话、四字大词堆叠）

判断标准：把任意一段抽出来读，能不能听出"有人在思考"。AI 生成的特征是过于流畅、对称、无逻辑摩擦；人类学术写作的特征是有节奏起伏、有判断保留、有具体细节锚定。

---

## 二、AI 痕迹三层模型（按检测难度递减）

### 第一层（最表层）：词汇与套话
- 易识别、易修改
- 但单纯改词汇收益有限，CNKI AIGC 检测器穿透表层

### 第二层：句式与标点
- 破折号插入语、不仅……更……、从 A 到 B
- 改起来需要重写句子

### 第三层（最深）：结构与论证模式
- 段落内部完美对称的并列
- 段尾必有"小结升华"句
- 论证链条无任何摩擦、转折、限定
- **这一层是 AI 检测真正捕获的特征，靠词表禁令解决不了**

修改顺序应反过来：**先治第三层，再治第二层，最后清第一层**。第一层修改是收尾工作，不要本末倒置。

---

## 三、第三层规则：结构与论证

### 3.1 段落内部允许摩擦

**反例**（AI 偏好的单向论证）：
> 美国通过《芯片法案》构建了"政策—组织—培养"的一体化链条。CHIPS Act 提供财政基础，NSTC 提供组织载体，技能联盟提供培养执行。三者协同推进，形成了完整的制度闭环，深度重塑了人才培养格局。

**正例**（带摩擦感的人类论证）：
> 美国通过《芯片法案》试图构建"政策—组织—培养"的一体化链条。CHIPS Act 提供财政基础，NSTC 承担组织协调，技能联盟落地培养执行。但三者协同并非自动达成：CHIPS 资金有明确产业指向，NSTC 内部对优先方向尚存分歧，技能联盟则在执行层面受地方学院能力限制。链条的"完整"更多体现在制度设计层面，运行效果尚待观察。

### 3.2 段尾不做"升华"

AI 段尾喜欢加一句概括性升华句（"这意味着……正在转向……"），人类不这么写。如果段尾确需收束，应该是 **一个具体的判断 + 一个限定**，而不是空洞的范式总结。

**反例**：
> 这意味着集成电路工程教育的治理重心正由院校内部改革转向国家主导的生态统筹。

**正例**：
> 治理重心的这一移动并不彻底——院校层面的课程改革仍由各高校自主推进，国家主导主要体现在跨机构平台的搭建与资金投放方向上。

### 3.3 案例并列段必须打破句式同构

**反例**（五个案例完美对称）：
> 美国以法案约束为手段，构建了……。日本以行政动员为路径，形成了……。韩国以财政集中为方式，建立了……。中国台湾地区以专项立法为基础，依托了……。欧盟以超国家协调为机制，发展了……。

**正例**（句式与切入角度差异化）：
> 美国选择了法案约束的路径：CHIPS Act 以条件化资助绑定企业培养承诺。日本则反向操作，将人才条款嵌入《半导体战略》而非单立法案，依靠经济产业省主导的省厅协调推进。韩国的特征是行政动员的高强度——尹锡悦政府连续以总统令推动半导体特区与定向招生。中国台湾地区采取了专项立法授权的方式，《重点领域产学合作及人才培育创新条例》打破了私立大学财务自主限制。欧盟则受困于教育主权分散，被迫走超国家协调的迂回路径。

注意：第二种写法不只是换了动词，而是 **句式长度、切入角度、判断密度都有变化**。

### 3.4 元叙述句的位置规则

元叙述句 = 告诉读者"本文/本节正在做什么"的句子（如"本文将……""下一节讨论……""如前所述……"）。

- 全文累计 **不超过 5 处**
- 集中在引言开头与结语开头
- 正文章节内部禁止使用"下文将……""上文已述……""本节首先……其次……"

如果一段话不用元叙述句也读得通，就删。

### 3.5 章节小结不做规范性收口

**反例**：
> 综上所述，政策维度的差异直接决定了组织维度的结构选择，进而约束了培养机制的实现路径。这一因果链条贯穿全文五个案例，构成本文的核心发现。

**正例**：
> 政策工具的差异并不机械决定组织选择——韩国的案例显示，行政动员可能在缺少法案约束的条件下达到相近效果。但工具与组织的耦合方向在五个案例中具有一致性：法案约束倾向于催生混合型机构，行政动员倾向于直接组建产业联盟。

## 四、第二层规则：句式与标点

### 4.1 破折号

**禁用**：句中插入语。
```
反例：CHIPS Act——美国近十年最重要的产业立法之一——明确将人才培养列为条件化条款。
正例：CHIPS Act 是美国近十年最重要的产业立法之一，其中明确将人才培养列为条件化条款。
```

**允许**：术语连接符（"政策—组织—培养"）、范围号（"12—28 纳米"）、英文摘要中的 em-dash。

### 4.2 递进句式

| 禁用 | 替代 |
|---|---|
| 不仅 X 更 Y | 不仅 X，也 Y |
| 不仅 X 更是 Y | X 与 Y 同时…… |
| X 不仅……更…… | X 既……又…… |

### 4.3 从 A 到 B

全文累计 ≤ 2 处。超出时改写：

| 原 | 改 |
|---|---|
| 从单一改革到系统重构 | 由单一改革推进至系统重构 |
| 从院校内部转向国家主导 | 治理重心由院校内部移至国家层面 |

### 4.4 句首禁语

```
值得注意的是 / 值得一提的是 / 不难发现 / 显而易见 / 不可否认
综上所述 / 综合以上分析 / 总而言之 / 总的来说
由此可见 / 不言而喻 / 一言以蔽之
```

全部禁用。需要表达类似意思时，直接给出判断，不加引导语。

### 4.5 形式主语与被动滥用

| 反例 | 正例 |
|---|---|
| 该方法被广泛使用 | 该方法应用广泛 |
| 它是重要的去考虑…… | 需要考虑…… |
| 在 2024 年，中国…… | 2024 年，中国…… |
| 由该政策所推动的改革 | 该政策推动的改革 |

## 五、第一层规则：词汇

详见 `H-banned-words.md`。本节只列原则：

### 5.1 硬禁用（绝不出现）

```
深度（作程度副词）、显著、赋能、恰恰、重塑（作动词）
```

### 5.2 密度管理（不消灭，控密度）

```
而非（全文 ≤ 5）、不仅...更（全文 ≤ 2）、这一（控频）、硬约束（表格内可、正文谨慎）、明显（控密度）
```

### 5.3 四字大词组合禁令

不允许"X 性 Y 化"或四字抽象叠加：
- 深度赋能、系统性重构、战略性重塑、全方位推进、根本性变革

### 5.4 立场词

- 用：笔者认为 / 笔者以为 / 本研究主张
- 不用：研究表明（除非真的引用其他研究）、本文认为（书面化但仍 AI 化）

## 六、术语与翻译选择

### 6.1 关键术语的中英对照（仅供检查时核对）

| 中文 | 英文 | 备注 |
|---|---|---|
| 战略性重构 | strategic institutional restructuring | 概念名 |
| 法案约束 / 条件化约束 | legislative conditionality | 跨国比较语境 |
| 政策工具组合 | policy instrument mix / policy mix | Rogge & Reichardt |
| 三螺旋理论 | Triple Helix | Etzkowitz & Leydesdorff |
| 国家创新体系 | National Innovation System / NIS | Freeman, Lundvall |
| 资本主义多样性 | Varieties of Capitalism / VoC | Hall & Soskice |
| 技能形成制度 | skill formation institutions | Thelen, Busemeyer |
| 中国台湾地区 | China's Taiwan region | 涉台合规 |

### 6.2 同一概念全文统一

- 首次出现时给出中英对照
- 后续使用统一中文译名
- 不要在同一文中混用"集成电路"和"半导体"——主称选定一个，国际语境固定搭配可保留另一称呼

## 七、自评命令清单（修改完一遍后扫一遍）

```bash
# 第一层：词汇
grep -nE '深度[^度学]|显著|赋能|恰恰|重塑[^力]' <file>
grep -cE '而非' <file>           # 应 ≤ 5
grep -cE '不仅.{0,20}更' <file>   # 应 ≤ 2
grep -cE '从.{1,10}到' <file>     # 应 ≤ 2

# 第二层：句式
grep -nE '值得注意|不难发现|显而易见|综上所述|综合以上|由此可见|不言而喻' <file>
grep -nE '——' <file>             # 人工检查每处是否插入语

# 第三层：结构
grep -cE '本文|本节|下文将|上文已' <file>   # 全文 ≤ 5
grep -nE '研究表明|本文认为' <file>          # 应改为"笔者认为"

# 全角标点合规
grep -nE '[,.;:?!]' <file> | grep -vE '\[.*\]|http|@'   # 半角标点漏网
```

## 八、修改流程

每次拿到一段需修改的文本，按以下顺序：

1. 通读一遍，判断有没有第三层问题（结构对称、单向论证、段尾升华）。有就 **重写**，不要逐句改。
2. 没有第三层问题时，扫第二层（破折号、递进句式、句首禁语）。逐句替换。
3. 最后扫第一层（词汇、立场词）。grep + 替换。
4. 改完通读一遍：是否还像"一个人在思考"？不像就重来。

不要顺序倒过来。词汇先改，结构没动，等于白做。
